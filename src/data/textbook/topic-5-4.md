## 概要

ネットワークデバイスの管理や設定変更を行うためには、デバイスに「アクセスする」方法を知っている必要があります。アクセス方法には、直接的な接続（コンソール）からリモートアクセス（SSH、Telnet、RDP、VPN）、さらにクラウドベースの管理（Meraki）まで、さまざまな選択肢があります。CCST 試験では、各アクセス方法の特徴、適切な使い分け、セキュリティ上の考慮事項が問われます。

## コンソール接続

<KeyTerm term="Console Connection" reading="コンソールセツゾク">ネットワークデバイスに物理的にケーブルで接続してアクセスする方法です。コンソールケーブル（ロールオーバーケーブル）を使用し、デバイスのコンソールポートと PC のシリアルポート（または USB ポート）を接続します。</KeyTerm>

コンソール接続は、ネットワークデバイスへの最も基本的なアクセス方法です。

**コンソール接続の特徴:**

- ネットワーク接続が不要（デバイスの IP アドレスが設定されていなくてもアクセス可能）
- 初期設定、パスワード回復、トラブルシューティングに使用
- 物理的にデバイスの近くにいる必要がある
- 通信は暗号化されない（ただしローカル接続のためリスクは低い）
- ターミナルエミュレーターソフト（PuTTY、Tera Term など）を使用
- デフォルト設定: 9600 bps、8 データビット、パリティなし、1 ストップビット

コンソール接続は、デバイスに初めて設定を投入する場合や、SSH/Telnet でアクセスできなくなった場合の最後の手段として特に重要です。

<KeyPoint>
- コンソール接続はネットワーク不要でデバイスに直接アクセスできる唯一の方法
- 初期設定やパスワード回復に必須
- 物理的にデバイスの近くにいる必要がある
</KeyPoint>

## SSH（Secure Shell）

<KeyTerm term="SSH" reading="エスエスエイチ">Secure Shell の略。ネットワーク経由でリモートデバイスに安全にアクセスするためのプロトコルです。すべての通信が暗号化されるため、パスワードやコマンドが傍受されるリスクがありません。ポート番号は TCP 22 を使用します。</KeyTerm>

SSH は、ネットワークデバイスへのリモートアクセスで最も推奨される方法です。

**SSH の特徴:**

- すべての通信（認証情報を含む）が暗号化される
- ユーザー名とパスワード、または公開鍵認証で認証する
- TCP ポート 22 を使用
- SSH バージョン 2（SSHv2）が推奨される（v1 は脆弱性あり）
- Cisco デバイスでは `ssh -l [ユーザー名] [IPアドレス]` でアクセス

**SSH の接続手順:**

1. PC にSSH クライアントソフトを用意（PuTTY、Windows Terminal、macOS/Linux のターミナルなど）
2. 接続先のデバイスの IP アドレスとポート 22 を指定
3. ユーザー名とパスワードを入力して認証
4. CLI（コマンドラインインターフェース）が表示され、デバイスの設定や確認が可能になる

## Telnet

<KeyTerm term="Telnet" reading="テルネット">ネットワーク経由でリモートデバイスの CLI にアクセスするためのプロトコルです。SSH と異なり、通信は暗号化されないため、セキュリティ上のリスクがあります。ポート番号は TCP 23 を使用します。</KeyTerm>

Telnet は SSH が普及する以前から使われていたリモートアクセスプロトコルです。

**Telnet の問題点:**

- 通信がプレーンテキスト（平文）で送信される
- パスワードやコマンドが傍受される危険性がある
- 現在はセキュリティ上の理由から**非推奨**
- 多くの組織で SSH への移行が進んでいる

**SSH と Telnet の比較:**

| 項目 | SSH | Telnet |
|---|---|---|
| ポート番号 | TCP 22 | TCP 23 |
| 暗号化 | あり | なし |
| セキュリティ | 高い | 低い |
| 推奨度 | 推奨 | 非推奨 |
| 認証 | ユーザー名/パスワードまたは公開鍵 | ユーザー名/パスワード |

<KeyPoint>
- SSH（ポート 22）は暗号化あり、Telnet（ポート 23）は暗号化なし
- ネットワークデバイスへのリモート CLI アクセスには SSH を使用すべき
- Telnet は平文通信のためセキュリティリスクがあり非推奨
</KeyPoint>

## RDP（Remote Desktop Protocol）

<KeyTerm term="RDP" reading="アールディーピー">Remote Desktop Protocol の略。Microsoft が開発したプロトコルで、リモートの Windows コンピューターのデスクトップ画面に接続し、GUI（グラフィカルユーザーインターフェース）で操作できます。ポート番号は TCP 3389 を使用します。</KeyTerm>

RDP は、Windows サーバーやデスクトップ PC にリモートからGUI でアクセスするために使用されます。

**RDP の特徴:**

- Windows のデスクトップ画面をリモートで操作可能
- TCP ポート 3389 を使用
- 通信は暗号化される
- Windows に標準搭載（「リモートデスクトップ接続」アプリ）
- 主に Windows サーバーの管理に使用される

RDP は CLI ではなく GUI でアクセスする点が SSH/Telnet との大きな違いです。サーバー管理では RDP が便利ですが、Cisco のネットワークデバイス（ルーター、スイッチ）は通常 CLI で管理するため、SSH やコンソールを使用します。

## VPN（Virtual Private Network）

<KeyTerm term="VPN" reading="ブイピーエヌ">Virtual Private Network の略。インターネットなどのパブリックネットワーク上に暗号化されたトンネルを作成し、あたかもプライベートネットワークに直接接続しているかのようにリモートアクセスする技術です。</KeyTerm>

VPN は、リモートの場所から社内ネットワークに安全にアクセスするために使用されます。

**VPN の主な種類:**

| 種類 | 説明 |
|---|---|
| リモートアクセス VPN | 個々のユーザーが自宅や外出先から社内ネットワークに接続 |
| サイト間 VPN | 異なる拠点のネットワーク同士を恒久的に接続 |

VPN を使用すると、リモートからでもあたかも社内ネットワークに直接接続しているかのようにネットワークデバイスにアクセスできます。VPN 接続後に SSH でスイッチやルーターに接続する、という使い方が一般的です。

<KeyPoint>
- RDP（ポート 3389）は Windows サーバーの GUI リモート管理に使用
- VPN は暗号化トンネルを通じてリモートから安全に社内ネットワークにアクセスする技術
- VPN 接続後に SSH でネットワークデバイスを管理するのが一般的な運用方法
</KeyPoint>

## NMS（Network Management System）

<KeyTerm term="NMS" reading="エヌエムエス">Network Management System の略。ネットワーク全体のデバイスを一元的に監視・管理するためのソフトウェアです。SNMP（Simple Network Management Protocol）を使用してデバイスの情報を収集し、ダッシュボードで可視化します。</KeyTerm>

NMS は、多数のネットワークデバイスを効率的に管理するためのツールです。

**NMS の主な機能:**

- ネットワークデバイスの状態監視（稼働状況、CPU 使用率、トラフィック量）
- アラート通知（デバイスダウン、閾値超過など）
- 設定管理（設定バックアップ、一括変更）
- パフォーマンス分析（帯域幅利用率、エラー率のグラフ化）
- ネットワークトポロジーの自動検出と可視化

代表的な NMS には、Cisco Prime Infrastructure、SolarWinds、Nagios、Zabbix などがあります。

## Cisco Meraki

<KeyTerm term="Cisco Meraki" reading="シスコメラキ">Cisco が提供するクラウドベースのネットワーク管理プラットフォームです。ルーター、スイッチ、アクセスポイント、セキュリティ機器をすべてクラウドのダッシュボードから一元管理できます。</KeyTerm>

Meraki は従来のオンプレミス型管理とは異なるアプローチのネットワーク管理ソリューションです。

**Meraki の特徴:**

- **クラウドベース**: Web ブラウザからダッシュボードにアクセスして管理
- **ゼロタッチプロビジョニング**: デバイスをネットワークに接続するだけで自動的に設定が適用される
- **一元管理**: 複数拠点のデバイスを 1 つのダッシュボードで管理
- **リアルタイム監視**: デバイスの状態、トラフィック、ユーザーをリアルタイムで監視
- **自動アップデート**: ファームウェアの更新がクラウドから自動配信

Meraki は CLI の知識がなくても GUI で直感的に管理できるため、専任のネットワークエンジニアがいない小規模拠点の管理に適しています。

## まとめ

- コンソール接続はネットワーク不要でデバイスに直接アクセスでき、初期設定やパスワード回復に必須です
- SSH（ポート 22）は暗号化された安全なリモートアクセス方法で、Telnet（ポート 23）の代わりに使用すべきです
- RDP（ポート 3389）は Windows の GUI リモートアクセスに使用されます
- VPN はリモートから社内ネットワークに安全に接続する技術で、SSH と組み合わせて使用します
- Cisco Meraki はクラウドベースのネットワーク管理プラットフォームで、Web ブラウザから一元管理が可能です
