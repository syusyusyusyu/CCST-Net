## 概要

ルーティングとは、ネットワーク上でパケットを宛先まで転送するプロセスです。ルーターはルーティングテーブルに基づいて、パケットをどのインターフェースから送出するかを決定します。このトピックでは、デフォルトゲートウェイ、ローカルネットワークとリモートネットワークの違い、ルーティングテーブルの基本について学びます。これらはネットワーク通信の根幹であり、CCST 試験でも重要な出題範囲です。

## デフォルトゲートウェイ

<KeyTerm term="Default Gateway" reading="デフォルトゲートウェイ">ローカルネットワーク上のデバイスが、他のネットワーク（インターネットなど）にパケットを送信する際に使用するルーターの IP アドレスです。ローカルネットワークの「出口」の役割を果たします。</KeyTerm>

すべてのエンドデバイス（PC、スマートフォン、サーバーなど）は、ローカルネットワーク外と通信するためにデフォルトゲートウェイを設定する必要があります。デフォルトゲートウェイは通常、ローカルネットワークに接続されたルーターの LAN 側インターフェースの IP アドレスです。

例えば、ネットワーク 192.168.1.0/24 に接続された PC の場合、デフォルトゲートウェイは 192.168.1.1（ルーターの LAN 側 IP アドレス）に設定されるのが一般的です。

**デフォルトゲートウェイが設定されていない場合、または間違っている場合:**
- ローカルネットワーク内のデバイスとは通信できます
- ローカルネットワーク外のデバイス（インターネット上のサーバーなど）とは通信できません
- `ping` でデフォルトゲートウェイに到達できるかを確認することが、トラブルシューティングの基本手順です

<KeyPoint>
- デフォルトゲートウェイはローカルネットワーク外への通信の「出口」
- デフォルトゲートウェイが未設定だと、ローカル外との通信ができない
- トラブルシューティングでは、まずデフォルトゲートウェイへの ping を試みる
</KeyPoint>

## ローカルネットワークとリモートネットワーク

ルーティングを理解するためには、ローカルネットワークとリモートネットワークの概念を理解することが重要です。

<KeyTerm term="Local Network" reading="ローカルネットワーク">送信元デバイスと同じサブネットに属するネットワークです。ローカルネットワーク内の通信はルーターを介さず、スイッチによって直接転送されます。</KeyTerm>

<KeyTerm term="Remote Network" reading="リモートネットワーク">送信元デバイスとは異なるサブネットに属するネットワークです。リモートネットワークへの通信は必ずルーターを経由する必要があります。</KeyTerm>

**ローカル / リモートの判定方法:**

デバイスは、宛先 IP アドレスが自分と同じネットワーク（サブネット）に属するかどうかを、自身の IP アドレスとサブネットマスクを使って判定します。

```
送信元 IP: 192.168.1.10
サブネットマスク: 255.255.255.0
→ ネットワークアドレス: 192.168.1.0

宛先 IP: 192.168.1.50  → 同じネットワーク → ローカル（直接送信）
宛先 IP: 10.0.0.5      → 異なるネットワーク → リモート（ゲートウェイへ送信）
```

- **ローカル通信**: ARP（Address Resolution Protocol）で宛先の MAC アドレスを解決し、スイッチ経由で直接フレームを送信します
- **リモート通信**: パケットをデフォルトゲートウェイ（ルーター）に送信し、ルーターが宛先ネットワークに向けて転送します

<KeyPoint>
- 同じサブネット内 = ローカル通信（ルーター不要、スイッチで転送）
- 異なるサブネット = リモート通信（ルーター経由、デフォルトゲートウェイに送信）
- 判定は送信元の IP アドレスとサブネットマスクから行われる
</KeyPoint>

## ルーターの役割

<KeyTerm term="Router" reading="ルーター">異なるネットワーク（サブネット）間でパケットを転送するネットワークデバイスです。OSI モデルの第 3 層（ネットワーク層）で動作し、IP アドレスに基づいてルーティング（経路選択）を行います。</KeyTerm>

ルーターの主な役割は以下のとおりです。

1. **パケット転送**: 宛先 IP アドレスに基づいて、パケットを適切なインターフェースから送出します
2. **ネットワーク間の接続**: 異なるサブネットやネットワークを相互に接続します
3. **最適経路の選択**: 複数の経路がある場合、最適な経路を選択してパケットを転送します
4. **ブロードキャストの分割**: ルーターはブロードキャストフレームを転送しないため、ブロードキャストドメインを分割します

## ルーティングテーブル

<KeyTerm term="Routing Table" reading="ルーティングテーブル">ルーターが保持するネットワークの経路情報のデータベースです。宛先ネットワーク、サブネットマスク、ネクストホップ（次に転送するルーターの IP アドレス）、出力インターフェースなどの情報が含まれます。</KeyTerm>

ルーターは、パケットを受信するたびにルーティングテーブルを参照して、パケットの転送先を決定します。ルーティングテーブルには以下の情報が含まれます。

| 項目 | 説明 |
|---|---|
| 宛先ネットワーク | パケットの宛先となるネットワークアドレス（例: 10.0.0.0/8） |
| ネクストホップ | パケットを次に転送するルーターの IP アドレス |
| 出力インターフェース | パケットを送出するルーターのインターフェース（例: Gi0/1） |
| メトリック / コスト | 経路の優先度を決める数値（小さいほど優先） |
| 経路の取得方法 | 直接接続、静的ルート、動的ルーティングプロトコル |

## 経路の種類

ルーティングテーブルに登録される経路には、以下の 3 つの種類があります。

**1. 直接接続ルート (Connected Route)**

ルーターのインターフェースに IP アドレスが設定され、そのインターフェースが有効化されると、自動的にルーティングテーブルに追加されます。ルーターが直接接続しているネットワークを示します。

**2. スタティックルート (Static Route)**

<KeyTerm term="Static Route" reading="スタティックルート">ネットワーク管理者が手動でルーティングテーブルに追加する経路情報です。小規模ネットワークや、特定の経路を明示的に指定したい場合に使用します。</KeyTerm>

管理者が `ip route` コマンドを使って手動で設定します。ネットワーク構成が変更された場合、管理者が手動で経路を更新する必要があります。

**3. ダイナミックルート (Dynamic Route)**

<KeyTerm term="Dynamic Routing Protocol" reading="ダイナミックルーティングプロトコル">ルーター同士が自動的にネットワーク情報を交換し、ルーティングテーブルを更新するプロトコルです。OSPF、EIGRP、RIP などがあります。</KeyTerm>

ルーティングプロトコル（OSPF、EIGRP など）を使って、ルーター間で自動的に経路情報を交換します。ネットワーク構成の変更に自動で対応できるため、大規模ネットワークで使用されます。

<KeyPoint>
- ルーティングテーブルには直接接続、スタティック、ダイナミックの 3 種類の経路がある
- 直接接続ルートはインターフェースの設定で自動追加される
- スタティックルートは管理者が手動で追加する
- ダイナミックルートはルーティングプロトコルで自動学習される
</KeyPoint>

## デフォルトルート

<KeyTerm term="Default Route" reading="デフォルトルート">ルーティングテーブルに一致する経路がない場合に使用される「最後の手段」の経路です。宛先ネットワークを 0.0.0.0/0 として設定します。</KeyTerm>

デフォルトルートは、特定の宛先への経路がルーティングテーブルに見つからなかった場合に使用されます。一般的にはインターネットへの接続に使用され、ISP（インターネットサービスプロバイダー）のルーターをネクストホップとして設定します。

Cisco ルーターでのデフォルトルートの設定例:

```
ip route 0.0.0.0 0.0.0.0 203.0.113.1
```

この設定は「すべての未知の宛先のパケットを 203.0.113.1 に転送する」という意味です。

## パケット転送の流れ

PC がインターネット上のサーバーと通信する場合のパケット転送の流れを確認しましょう。

1. PC は宛先 IP アドレスが自分のサブネット外であることを判定します
2. PC はパケットをデフォルトゲートウェイ（ルーター）に送信します
3. ルーターはルーティングテーブルを参照し、宛先ネットワークへの最適な経路を見つけます
4. ルーターはパケットをネクストホップのルーターまたは直接接続されたネットワークに転送します
5. このプロセスが宛先ネットワークに到達するまで繰り返されます

各ルーターでは、パケットの宛先 IP アドレスは変わりませんが、フレームの MAC アドレスは各ホップで書き換えられます。これを「ホップバイホップルーティング」と呼びます。

## まとめ

- デフォルトゲートウェイは、エンドデバイスがリモートネットワークに通信するための出口となるルーターの IP アドレスです
- ローカル通信（同じサブネット内）はルーター不要、リモート通信（異なるサブネット）はルーターが必要です
- ルーティングテーブルには直接接続、スタティック、ダイナミックの 3 種類の経路が含まれます
- デフォルトルート（0.0.0.0/0）は、一致する経路がない場合の「最後の手段」の経路です
- パケットはルーターのルーティングテーブルに基づいてホップバイホップで転送されます
