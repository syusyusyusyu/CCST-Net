## 概要

ネットワークに接続するためには、各デバイスで正しいネットワーク設定が行われている必要があります。このトピックでは、Windows、Linux、macOS、Android、iOS の各プラットフォームにおけるネットワーク設定の確認方法と、IP アドレスの取得・設定手順について学びます。CCST 試験では、OS ごとの設定画面やコマンドの違いが問われることがあるため、しっかり押さえておきましょう。

## ネットワーク設定の基本要素

デバイスがネットワークに正しく接続するためには、以下の情報が必要です。

<KeyTerm term="IP Address" reading="アイピーアドレス">ネットワーク上でデバイスを一意に識別するための番号です。IPv4 では 192.168.1.10 のような形式で表されます。</KeyTerm>

<KeyTerm term="Subnet Mask" reading="サブネットマスク">IP アドレスのうち、ネットワーク部とホスト部を区別するためのビットマスクです。例えば 255.255.255.0 はネットワーク部が 24 ビットであることを示します。</KeyTerm>

<KeyTerm term="Default Gateway" reading="デフォルトゲートウェイ">ローカルネットワーク外の宛先にパケットを転送する際に使用するルーターの IP アドレスです。通常はルーターの LAN 側アドレスが設定されます。</KeyTerm>

<KeyTerm term="DNS Server" reading="ディーエヌエスサーバー">ドメイン名（例: www.example.com）を IP アドレスに変換する名前解決サービスを提供するサーバーです。</KeyTerm>

これらの設定は、DHCP によって自動的に取得されるか、手動（静的）で設定するかのいずれかです。

<KeyPoint>
- ネットワーク接続に必要な 4 つの情報: IP アドレス、サブネットマスク、デフォルトゲートウェイ、DNS サーバー
- DHCP を使えば自動取得、手動設定の場合は静的 IP を入力する
- 設定が正しくないとネットワークに接続できない、またはインターネットにアクセスできない
</KeyPoint>

## DHCP と静的 IP アドレス

ほとんどの家庭や企業のネットワークでは、<KeyTerm term="DHCP" reading="ディーエイチシーピー">Dynamic Host Configuration Protocol の略。ネットワーク上のデバイスに IP アドレス、サブネットマスク、デフォルトゲートウェイ、DNS サーバーを自動的に割り当てるプロトコルです。</KeyTerm> を使用してネットワーク設定を自動取得します。

DHCP を使う場合、デバイスはネットワークに接続すると自動的に DHCP サーバー（通常はルーター）から IP アドレスなどの設定を受け取ります。ユーザーが手動で設定する必要はありません。

一方、サーバーやプリンターなど、常に同じ IP アドレスを使用する必要があるデバイスには、静的 IP アドレスを手動で設定します。静的設定の場合、IP アドレス、サブネットマスク、デフォルトゲートウェイ、DNS サーバーをすべて手動で入力する必要があります。

## Windows でのネットワーク設定

Windows では、以下の方法でネットワーク設定を確認・変更できます。

**GUI での確認方法:**

1. 「設定」→「ネットワークとインターネット」→「状態」を開きます
2. 「プロパティ」をクリックすると、現在の接続情報が表示されます
3. IP アドレス、サブネットマスク、デフォルトゲートウェイ、DNS サーバーが確認できます

**コマンドでの確認方法:**

コマンドプロンプトで `ipconfig` コマンドを実行すると、すべてのネットワークアダプターの情報が表示されます。`ipconfig /all` を使うと、DHCP サーバーのアドレスや MAC アドレスなどの詳細情報も確認できます。

`ipconfig /release` で DHCP リースを解放し、`ipconfig /renew` で新しい IP アドレスを再取得できます。

## Linux でのネットワーク設定

Linux では、主にコマンドラインでネットワーク設定を確認します。

**ip コマンド（推奨）:**

`ip addr show` または `ip a` で、すべてのネットワークインターフェースの IP アドレスが表示されます。`ip route show` でルーティングテーブル（デフォルトゲートウェイ含む）を確認できます。

**ifconfig コマンド（レガシー）:**

古い Linux ディストリビューションでは `ifconfig` コマンドが使用されていました。現在は `ip` コマンドが推奨されていますが、試験では両方のコマンドが問われる可能性があります。

<KeyPoint>
- Windows: `ipconfig` コマンドで IP アドレスを確認
- Linux: `ip addr show` または `ifconfig` で確認（`ip` コマンドが推奨）
- macOS: `ifconfig` コマンドまたはシステム環境設定で確認
</KeyPoint>

## macOS でのネットワーク設定

macOS では、GUI とコマンドラインの両方で確認できます。

**GUI での確認:**

「システム環境設定」（または「システム設定」）→「ネットワーク」を開きます。接続中のネットワークインターフェースを選択すると、IP アドレス、サブネットマスク、ルーター（デフォルトゲートウェイ）が表示されます。

**コマンドラインでの確認:**

ターミナルで `ifconfig` コマンドを使用します。macOS は BSD 系の UNIX をベースにしているため、Linux の `ip` コマンドではなく `ifconfig` を使用します。

## モバイルデバイスでの設定

**Android:**

「設定」→「ネットワークとインターネット」→「Wi-Fi」から接続中のネットワークをタップし、「詳細」を展開すると IP アドレスやゲートウェイが確認できます。静的 IP の設定は、同じ画面で「IP 設定」を「静的」に変更することで行えます。

**iOS (iPhone/iPad):**

「設定」→「Wi-Fi」から接続中のネットワーク名の右にある「i」アイコンをタップします。IP アドレス、サブネットマスク、ルーター（デフォルトゲートウェイ）が表示されます。「IP を構成」から DHCP と手動（静的）を切り替えられます。

<KeyPoint>
- Android: 設定 → Wi-Fi → 接続中のネットワークの詳細で確認
- iOS: 設定 → Wi-Fi → 「i」アイコンで確認
- モバイルデバイスでも DHCP と静的 IP の切り替えが可能
</KeyPoint>

## トラブルシューティングへの活用

ネットワーク設定の確認は、接続トラブルの原因を特定する際に最初に行うべき手順です。

- **IP アドレスが 169.254.x.x になっている場合**: DHCP サーバーから IP アドレスを取得できていません。これは APIPA（Automatic Private IP Addressing）のアドレスです。ケーブル接続や DHCP サーバーの状態を確認しましょう。
- **IP アドレスは正しいがインターネットに接続できない場合**: デフォルトゲートウェイや DNS サーバーの設定を確認します。
- **特定のサイトにアクセスできない場合**: DNS サーバーの設定や、`nslookup` コマンドで名前解決が正しく行われているかを確認します。

## まとめ

- ネットワーク接続には IP アドレス、サブネットマスク、デフォルトゲートウェイ、DNS サーバーの 4 つの設定が必要です
- Windows では `ipconfig`、Linux では `ip addr show` または `ifconfig`、macOS では `ifconfig` でネットワーク設定を確認できます
- DHCP を使えば自動設定が可能で、静的 IP は手動入力が必要です
- モバイルデバイス（Android / iOS）でも Wi-Fi 設定画面からネットワーク情報を確認できます
- IP アドレスが 169.254.x.x の場合は DHCP の取得に失敗しており、APIPA アドレスが割り当てられた状態です
