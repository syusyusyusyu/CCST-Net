## 概要

IPv4 アドレスの枯渇問題を根本的に解決するために開発されたのが IPv6 です。このトピックでは、IPv6 アドレスの構造と表記法、プレフィックスの概念、そして IPv4 との主な違いについて学びます。IPv6 は今後のネットワーク基盤として普及が進んでおり、CCST 試験でも基本的な知識が求められます。

## IPv6 の必要性

IPv4 アドレスは 32 ビットで約 43 億個のアドレスしか表現できません。インターネットに接続するデバイスの急増（スマートフォン、IoT 機器など）により、IPv4 アドレスは世界的に枯渇しています。

NAT やプライベートアドレスで一時的に延命できていますが、根本的な解決策として **128 ビット** のアドレス空間を持つ IPv6 が設計されました。IPv6 のアドレス総数は約 3.4 × 10^38 個という天文学的な数であり、事実上無限のアドレスを提供できます。

## IPv6 アドレスの構造

<KeyTerm term="IPv6 Address" reading="アイピーブイシックスアドレス">128 ビットのアドレスで、16 ビットごとにコロン（:）で区切り、16 進数で表記します。</KeyTerm>

IPv6 アドレスは 128 ビットの数値を 16 ビットずつ 8 つのグループに分け、各グループを 4 桁の 16 進数で表記し、コロン（`:`）で区切ります。

```
完全表記: 2001:0db8:0000:0000:0000:0000:0000:0001
```

### IPv6 アドレスの省略規則

IPv6 アドレスは長いため、2 つの省略規則が定められています。

**規則 1: 各グループの先頭の 0 を省略できる**

```
2001:0db8:0000:0000:0000:0000:0000:0001
↓
2001:db8:0:0:0:0:0:1
```

各グループの先頭にある連続した 0 は省略できますが、各グループに最低 1 桁は残す必要があります（`0000` → `0`）。

**規則 2: 連続する 0 のグループを `::` に短縮できる（1 回のみ）**

```
2001:db8:0:0:0:0:0:1
↓
2001:db8::1
```

`::` は「ここに 0 のグループが連続している」ことを示します。ただし、アドレスの復元が一意にできるように、**`::` は 1 つのアドレス内で 1 回だけ** 使用できます。

**省略の具体例:**

| 完全表記 | 省略表記 |
|---|---|
| 2001:0db8:0000:0000:0000:0000:0000:0001 | 2001:db8::1 |
| fe80:0000:0000:0000:0211:22ff:fe33:4455 | fe80::211:22ff:fe33:4455 |
| 0000:0000:0000:0000:0000:0000:0000:0001 | ::1 |
| 0000:0000:0000:0000:0000:0000:0000:0000 | :: |

<KeyPoint>
- IPv6 アドレスは 128 ビットで、16 ビットずつ 8 グループ、コロン区切りの 16 進数で表記する
- 先頭の 0 は省略可能（0db8 → db8、0000 → 0）
- 連続する 0 のグループは `::` で省略可能だが、1 つのアドレス内で 1 回のみ
</KeyPoint>

## IPv6 プレフィックス

<KeyTerm term="IPv6 Prefix" reading="アイピーブイシックスプレフィックス">IPv4 のネットワーク部に相当する部分。プレフィックス長をスラッシュ表記（/64 など）で示します。</KeyTerm>

IPv6 では、IPv4 のサブネットマスクの代わりに **プレフィックス長** を使います。表記方法は CIDR と同じく、スラッシュのあとにプレフィックスのビット数を記述します。

```
2001:db8:abcd:1234::/64
```

この例では、先頭 64 ビットがプレフィックス（ネットワーク部）、残りの 64 ビットがインターフェース ID（ホスト部に相当）です。

### 一般的なプレフィックス長

| プレフィックス長 | 用途 |
|---|---|
| /48 | 組織（サイト）に割り当てられる標準的なプレフィックス |
| /64 | 1 つのサブネット（LAN）に割り当てられる標準的なプレフィックス |
| /128 | 1 つのホスト（ループバックなど） |

一般的なネットワーク設計では、**/64** が最も標準的なプレフィックス長として使用されます。

## IPv6 アドレスの種類

IPv6 にはいくつかの種類のアドレスがあります。

### ユニキャストアドレス

1 対 1 の通信に使用されるアドレスです。

- **グローバルユニキャストアドレス**: IPv4 のパブリックアドレスに相当。インターネット上でルーティング可能。`2000::/3` の範囲（2 または 3 で始まるアドレス）
- **リンクローカルアドレス**: 同一リンク（LAN）内でのみ有効なアドレス。`fe80::/10` の範囲。IPv6 が有効なすべてのインターフェースに自動的に割り当てられる

<KeyTerm term="Link-Local Address" reading="リンクローカルアドレス">fe80::/10 の範囲のアドレス。同一リンク内でのみ通信可能で、ルーターを越えた通信には使えません。IPv6 インターフェースに自動設定されます。</KeyTerm>

- **ユニークローカルアドレス**: IPv4 のプライベートアドレスに相当。`fc00::/7` の範囲（実際には fd00::/8 が使用される）。組織内部でのみ使用

### マルチキャストアドレス

<KeyTerm term="Multicast Address" reading="マルチキャストアドレス">特定のグループに属する複数のデバイスに同時にデータを送信するためのアドレス。IPv6 では ff00::/8 の範囲です。</KeyTerm>

IPv6 のマルチキャストアドレスは `ff00::/8`（ff で始まる）の範囲です。IPv4 のブロードキャスト（すべてのデバイスへの一斉送信）は IPv6 では廃止され、代わりにマルチキャストが使用されます。

### エニーキャストアドレス

複数のデバイスに同じアドレスを割り当て、最も近いデバイスが応答する仕組みです。ユニキャストアドレスと同じ形式ですが、複数のインターフェースに同じアドレスが設定されます。

<KeyPoint>
- グローバルユニキャスト（2000::/3）はインターネット上で使用、リンクローカル（fe80::/10）は LAN 内専用
- IPv6 にはブロードキャストがなく、代わりにマルチキャスト（ff00::/8）が使用される
- すべての IPv6 インターフェースにはリンクローカルアドレスが自動的に設定される
</KeyPoint>

## IPv4 と IPv6 の主な違い

| 項目 | IPv4 | IPv6 |
|---|---|---|
| アドレス長 | 32 ビット | 128 ビット |
| アドレス数 | 約 43 億 | 約 3.4 × 10^38 |
| 表記法 | 10 進数、ドット区切り | 16 進数、コロン区切り |
| ヘッダー | 可変長（20〜60 バイト） | 固定長（40 バイト） |
| ブロードキャスト | あり | **なし**（マルチキャストで代替） |
| アドレス設定 | 手動 or DHCP | **SLAAC** or DHCPv6 |
| NAT | 広く使用 | 基本的に不要（アドレスが豊富） |
| IPsec | オプション | 標準でサポート |

### SLAAC（Stateless Address Autoconfiguration）

<KeyTerm term="SLAAC" reading="スラーク">IPv6 デバイスが DHCP サーバーなしで自動的に IP アドレスを設定する仕組み。ルーターからプレフィックス情報を取得し、インターフェース ID を自動生成します。</KeyTerm>

SLAAC は IPv6 の重要な特徴の 1 つです。デバイスはルーターから広告される **プレフィックス情報** と、自身の MAC アドレスなどから生成する **インターフェース ID** を組み合わせて、自動的にグローバルユニキャストアドレスを設定できます。

## 特殊な IPv6 アドレス

| アドレス | 用途 |
|---|---|
| ::1 | ループバックアドレス（IPv4 の 127.0.0.1 に相当） |
| :: | 未指定アドレス（IPv4 の 0.0.0.0 に相当） |
| ff02::1 | リンクローカル全ノードマルチキャスト（すべてのデバイス） |
| ff02::2 | リンクローカル全ルーターマルチキャスト（すべてのルーター） |

<KeyPoint>
- IPv6 は 128 ビットで事実上無限のアドレスを提供し、NAT が基本的に不要になる
- SLAAC により DHCP サーバーなしでアドレスを自動設定できる
- ::1 がループバック、:: が未指定アドレスであり、IPv4 との対応を覚えておく
</KeyPoint>

## まとめ

- **IPv6** は IPv4 アドレス枯渇の根本的な解決策として設計された 128 ビットのアドレス体系である
- IPv6 アドレスは **16 進数・コロン区切り** で表記し、**先頭の 0 の省略** と **:: による省略** の 2 つの短縮規則がある
- **プレフィックス長** （/64 が標準）で IPv4 のサブネットに相当するネットワーク部分を示す
- IPv6 には **ユニキャスト**（グローバル、リンクローカル、ユニークローカル）、**マルチキャスト**、**エニーキャスト** の 3 種類のアドレスがある
- IPv4 との主な違いとして、**ブロードキャストの廃止**、**SLAAC による自動設定**、**NAT が基本的に不要** などが挙げられる
