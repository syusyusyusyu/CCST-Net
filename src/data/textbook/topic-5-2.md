## 概要

Wireshark は、ネットワーク上を流れるパケットをキャプチャ（捕捉）して分析するための、世界で最も広く使われているオープンソースのプロトコルアナライザーです。ネットワークのトラブルシューティングやセキュリティ分析において非常に強力なツールです。CCST 試験では、Wireshark の基本操作、キャプチャファイルの保存・読み込み、パケットの基本的な解釈が問われます。

## Wireshark とは

<KeyTerm term="Wireshark" reading="ワイヤーシャーク">ネットワークインターフェースを流れるパケットをリアルタイムでキャプチャし、プロトコルごとに詳細を解析できるオープンソースのパケットアナライザーです。Windows、macOS、Linux で利用できます。</KeyTerm>

<KeyTerm term="Packet Capture" reading="パケットキャプチャ">ネットワークインターフェースを通過するデータ（パケット）を捕捉して記録することです。キャプチャしたデータを分析することで、通信の内容やネットワークの問題を調査できます。</KeyTerm>

Wireshark を使うと、以下のようなことが可能です。

- ネットワーク上の通信をリアルタイムで監視する
- 特定のプロトコルの動作を詳細に確認する
- ネットワーク障害の原因を調査する
- セキュリティ上の問題（不正な通信など）を検出する
- アプリケーションの通信動作をデバッグする

## パケットキャプチャの開始

Wireshark でパケットキャプチャを行う基本的な手順は以下のとおりです。

1. **Wireshark を起動する**: 起動画面にネットワークインターフェースの一覧が表示されます
2. **キャプチャするインターフェースを選択する**: 有線接続なら「Ethernet」や「イーサネット」、無線接続なら「Wi-Fi」を選択します。各インターフェースの横にはトラフィックの波形が表示され、通信が活発なインターフェースが分かります
3. **キャプチャを開始する**: インターフェースをダブルクリックするか、ツールバーの青いサメのひれアイコン（開始ボタン）をクリックします
4. **キャプチャを停止する**: ツールバーの赤い四角アイコン（停止ボタン）をクリックします

<KeyPoint>
- Wireshark の起動画面でキャプチャするインターフェースを選択する
- 通信が活発なインターフェースはトラフィック波形で確認できる
- 青いサメのひれアイコンでキャプチャ開始、赤い四角で停止
</KeyPoint>

## Wireshark の画面構成

パケットキャプチャ中の Wireshark の画面は、主に 3 つのペイン（領域）に分かれています。

**1. パケットリストペイン（上部）**

キャプチャされたパケットが 1 行 1 パケットで一覧表示されます。各行には以下の情報が表示されます。

| カラム | 内容 |
|---|---|
| No. | パケット番号（キャプチャした順番） |
| Time | キャプチャ開始からの経過時間 |
| Source | 送信元 IP アドレス |
| Destination | 宛先 IP アドレス |
| Protocol | プロトコル名（TCP、UDP、HTTP、DNS など） |
| Length | パケットの長さ（バイト数） |
| Info | パケットの概要情報 |

パケットはプロトコルごとに色分けされて表示されます。例えば、TCP は薄い紫色、UDP は薄い青色、HTTP は緑色、DNS は青色で表示されるのが一般的です。

**2. パケット詳細ペイン（中部）**

パケットリストで選択したパケットの詳細情報が、プロトコル層ごとにツリー形式で表示されます。各層を展開すると、ヘッダーフィールドの値を確認できます。

- Frame（フレーム情報）
- Ethernet II（イーサネットヘッダー: 送信元/宛先 MAC アドレス）
- Internet Protocol（IP ヘッダー: 送信元/宛先 IP アドレス、TTL など）
- Transmission Control Protocol / User Datagram Protocol（TCP/UDP ヘッダー: ポート番号など）
- アプリケーション層のデータ（HTTP、DNS など）

**3. パケットバイトペイン（下部）**

パケットの生データが 16 進数と ASCII 文字で表示されます。パケット詳細ペインでフィールドを選択すると、対応するバイトがハイライト表示されます。

<KeyPoint>
- パケットリストペイン（上）: キャプチャ済みパケットの一覧
- パケット詳細ペイン（中）: 選択パケットのプロトコル層ごとの詳細
- パケットバイトペイン（下）: パケットの生データ（16 進数表示）
</KeyPoint>

## 表示フィルター

キャプチャしたパケットが大量にある場合、表示フィルターを使って必要なパケットだけを表示できます。

<KeyTerm term="Display Filter" reading="ディスプレイフィルター">キャプチャ済みのパケットから特定の条件に合うものだけを表示するためのフィルターです。Wireshark 画面上部のフィルターバーに条件式を入力します。</KeyTerm>

**よく使う表示フィルターの例:**

| フィルター | 説明 |
|---|---|
| `ip.addr == 192.168.1.1` | 特定の IP アドレスを含むパケット |
| `tcp.port == 80` | TCP ポート 80 のパケット |
| `http` | HTTP プロトコルのパケットのみ |
| `dns` | DNS プロトコルのパケットのみ |
| `icmp` | ICMP（ping）パケットのみ |
| `tcp.flags.syn == 1` | TCP SYN フラグが設定されたパケット |
| `ip.src == 10.0.0.1` | 送信元が特定の IP アドレスのパケット |
| `ip.dst == 10.0.0.1` | 宛先が特定の IP アドレスのパケット |

フィルター式は `&&`（AND）、`||`（OR）、`!`（NOT）で組み合わせることもできます。

例: `ip.addr == 192.168.1.1 && tcp.port == 443` は、IP アドレス 192.168.1.1 かつ TCP ポート 443 のパケットを表示します。

## キャプチャフィルター

<KeyTerm term="Capture Filter" reading="キャプチャフィルター">パケットキャプチャの開始前に設定するフィルターです。条件に合うパケットのみをキャプチャし、不要なパケットを記録しないことでファイルサイズを抑え、分析を効率化します。</KeyTerm>

キャプチャフィルターは、表示フィルターとは構文が異なります（BPF: Berkeley Packet Filter 構文を使用）。

**よく使うキャプチャフィルターの例:**

| フィルター | 説明 |
|---|---|
| `host 192.168.1.1` | 特定のホストとの通信 |
| `port 80` | ポート 80 の通信 |
| `tcp` | TCP パケットのみ |
| `udp` | UDP パケットのみ |
| `not broadcast` | ブロードキャスト以外 |

キャプチャフィルターはキャプチャ開始後に変更できないため、あらかじめ適切な条件を設定しておく必要があります。

<KeyPoint>
- 表示フィルター: キャプチャ後に表示するパケットを絞り込む（いつでも変更可能）
- キャプチャフィルター: キャプチャ対象を事前に限定する（開始後は変更不可）
- 両者は構文が異なるため注意が必要
</KeyPoint>

## キャプチャファイルの保存と読み込み

キャプチャしたパケットデータは、ファイルとして保存し、後で再度分析できます。

**保存形式:**

- **pcapng（推奨）**: Wireshark のデフォルト形式。メタデータ（インターフェース情報、コメントなど）を含むことができます。拡張子は `.pcapng` です
- **pcap**: 従来の形式。互換性が高いですが、機能は pcapng より限定的です。拡張子は `.pcap` です

**保存方法:**

「ファイル」→「名前を付けて保存」（または Ctrl+Shift+S）でキャプチャファイルを保存します。特定のパケットだけをエクスポートしたい場合は、「ファイル」→「指定したパケットをエクスポート」を使用します。

**読み込み方法:**

「ファイル」→「開く」で保存済みのキャプチャファイルを開けます。他の人が保存したキャプチャファイルを受け取って分析することも可能です。これはチームでのトラブルシューティングにおいて非常に有用です。

## パケット分析の基本

Wireshark でパケットを分析する際のポイントをいくつか紹介します。

**TCP のスリーウェイハンドシェイクの確認:**

TCP 接続の確立は SYN → SYN-ACK → ACK の 3 つのパケットで行われます。この手順が正しく完了していない場合、接続に問題があることが分かります。

**DNS クエリと応答の確認:**

名前解決の問題を調査する際は、`dns` フィルターを使って DNS クエリ（要求）とレスポンス（応答）を確認します。クエリに対する応答がない場合、DNS サーバーに問題がある可能性があります。

**ICMP エラーメッセージの確認:**

`icmp` フィルターを使って、「Destination Unreachable」（宛先到達不能）や「Time Exceeded」（TTL 超過）などのエラーメッセージを確認できます。

## まとめ

- Wireshark はネットワークパケットをキャプチャ・分析するオープンソースの強力なツールです
- 画面は「パケットリスト」「パケット詳細」「パケットバイト」の 3 ペインで構成されます
- 表示フィルター（キャプチャ後）とキャプチャフィルター（キャプチャ前）を使い分けて効率的に分析します
- キャプチャファイルは pcapng 形式で保存・共有でき、チームでの分析に活用できます
- TCP ハンドシェイク、DNS クエリ、ICMP エラーなど、基本的なプロトコル動作を確認できます
