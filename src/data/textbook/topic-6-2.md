## 概要

ネットワークセキュリティを理解するうえで、基本的なセキュリティ概念の習得は欠かせません。このトピックでは、情報セキュリティの3つの柱である CIA トライアド、認証・認可・アカウンティングの AAA フレームワーク、多要素認証（MFA）、暗号化技術、Active Directory、そしてフィッシング・マルウェア・DoS といった代表的な脅威について学びます。これらの概念は CCST 試験で頻出するだけでなく、IT セキュリティの実務でも日常的に使われる基礎知識です。

## CIA トライアド

<KeyTerm term="CIA Triad" reading="シーアイエートライアド">情報セキュリティの3つの基本原則である機密性（Confidentiality）、完全性（Integrity）、可用性（Availability）をまとめた概念です。</KeyTerm>は、情報セキュリティのすべての基盤となるフレームワークです。

### 機密性（Confidentiality）

機密性とは、許可された人だけが情報にアクセスできるようにすることです。暗号化、アクセス制御、パスワード保護などの手段で実現します。例えば、患者の医療記録は担当医師と本人のみがアクセスできるべきです。

### 完全性（Integrity）

完全性とは、情報が不正に変更されていないことを保証することです。ハッシュ関数、デジタル署名、チェックサムなどの技術で検証します。例えば、銀行の送金データが途中で改ざんされていないことを確認する仕組みが必要です。

### 可用性（Availability）

可用性とは、許可されたユーザーが必要なときに情報やシステムにアクセスできることです。冗長化、バックアップ、UPS（無停電電源装置）などで実現します。サーバーがダウンしてサービスが利用できなくなることは、可用性の侵害にあたります。

<KeyPoint>
- CIA トライアドは情報セキュリティの最も基本的なフレームワーク
- 機密性 = 誰がアクセスできるか、完全性 = データが改ざんされていないか、可用性 = いつでもアクセスできるか
- CCST 試験では、具体的なシナリオがCIAのどの要素に該当するかを問う問題が出題される
</KeyPoint>

## AAA フレームワーク

<KeyTerm term="AAA" reading="トリプルエー">Authentication（認証）、Authorization（認可）、Accounting（アカウンティング）の3つのセキュリティ機能をまとめたフレームワークです。</KeyTerm>は、ネットワークリソースへのアクセス制御を体系化したフレームワークです。

### 認証（Authentication）

「あなたは誰ですか？」を確認するプロセスです。ユーザー名とパスワードの入力、生体認証、証明書の提示などが該当します。

### 認可（Authorization）

「あなたは何をできますか？」を決定するプロセスです。認証されたユーザーに対して、アクセスできるリソースや実行できる操作を制御します。例えば、一般社員は社内ポータルにアクセスできるが、サーバー管理コンソールにはアクセスできない、といった制御です。

### アカウンティング（Accounting）

「あなたは何をしましたか？」を記録するプロセスです。ログインの日時、アクセスしたリソース、実行した操作などを記録します。セキュリティ監査やインシデント調査に不可欠です。

AAA を実装する代表的なプロトコルとして、<KeyTerm term="RADIUS" reading="ラディウス">Remote Authentication Dial-In User Service の略。ネットワークアクセスの認証・認可・アカウンティングを一元管理するプロトコルです。</KeyTerm>と<KeyTerm term="TACACS+" reading="タカックスプラス">Terminal Access Controller Access-Control System Plus の略。Cisco が開発した AAA プロトコルで、認証・認可・アカウンティングを個別に制御できます。</KeyTerm>があります。

## 多要素認証（MFA）

<KeyTerm term="Multi-Factor Authentication" reading="たようそにんしょう">2つ以上の異なる認証要素を組み合わせて本人確認を行う認証方式です。MFA と略されます。</KeyTerm>は、セキュリティを大幅に向上させる認証方式です。

認証要素は以下の3つのカテゴリに分類されます。

| カテゴリ | 説明 | 例 |
|---|---|---|
| 知識要素（Something you know） | ユーザーが知っている情報 | パスワード、PIN、秘密の質問 |
| 所持要素（Something you have） | ユーザーが持っている物 | スマートフォン、セキュリティトークン、ICカード |
| 生体要素（Something you are） | ユーザーの身体的特徴 | 指紋、虹彩、顔認識、声紋 |

MFA では、これらの**異なるカテゴリから2つ以上**を組み合わせます。パスワードと PIN の組み合わせは、両方とも知識要素であるため、厳密には MFA とは呼びません。一方、パスワード（知識）とスマートフォンのワンタイムパスワード（所持）の組み合わせは、有効な MFA です。

<KeyPoint>
- MFA は「異なるカテゴリ」の認証要素を2つ以上組み合わせることが重要
- パスワード + PIN は同じカテゴリ（知識）なので MFA とは言えない
- パスワード + 指紋認証は異なるカテゴリ（知識 + 生体）なので有効な MFA
</KeyPoint>

## 暗号化

<KeyTerm term="Encryption" reading="あんごうか">データを特定のアルゴリズムと鍵を用いて変換し、許可されていない第三者が内容を読めないようにする技術です。</KeyTerm>は、機密性を実現するための核心的な技術です。

### 対称鍵暗号

暗号化と復号に**同じ鍵**を使用する方式です。処理速度が速いため、大量のデータの暗号化に適しています。代表的なアルゴリズムには AES（Advanced Encryption Standard）があります。課題は、鍵の安全な共有方法です。

### 非対称鍵暗号（公開鍵暗号）

暗号化と復号に**異なる鍵のペア**（公開鍵と秘密鍵）を使用する方式です。公開鍵で暗号化したデータは対応する秘密鍵でのみ復号できます。RSA が代表的なアルゴリズムです。鍵の配布問題を解決しますが、処理速度は対称鍵暗号より遅くなります。

### 実用での利用例

- **HTTPS**: Web ブラウザとサーバー間の通信を TLS/SSL で暗号化
- **VPN**: インターネット上にセキュアなトンネルを構築して通信を暗号化
- **SSH**: リモートアクセス時の通信を暗号化（Telnet の安全な代替）

## Active Directory

<KeyTerm term="Active Directory" reading="アクティブディレクトリ">Microsoft が開発したディレクトリサービスで、Windows ネットワーク環境においてユーザーアカウント、コンピュータ、グループなどのリソースを一元管理するシステムです。AD と略されます。</KeyTerm>は、企業ネットワークにおけるアイデンティティ管理の中核です。

Active Directory の主な機能は以下のとおりです。

- **ユーザーアカウントの一元管理**: 1つのアカウントで複数のリソースにアクセス（シングルサインオン）
- **グループポリシー**: セキュリティ設定やソフトウェアのインストールを一括で管理・適用
- **組織単位（OU）**: 部門やチームごとにリソースを整理し、異なるポリシーを適用
- **LDAP プロトコル**: ディレクトリ情報の検索・更新に使用される標準プロトコル

## 代表的なセキュリティ脅威

### フィッシング（Phishing）

<KeyTerm term="Phishing" reading="フィッシング">正規の組織や個人になりすまして、メールや偽の Web サイトを通じてユーザーの個人情報（パスワード、クレジットカード番号など）を不正に取得しようとするソーシャルエンジニアリング攻撃です。</KeyTerm>は、最も広く行われているサイバー攻撃の1つです。

フィッシングの種類には以下のものがあります。

- **スピアフィッシング**: 特定の個人や組織を狙った標的型攻撃
- **ホエーリング**: 経営幹部など高い権限を持つ人物を狙った攻撃
- **スミッシング**: SMS（ショートメッセージ）を使用した攻撃

### マルウェア（Malware）

<KeyTerm term="Malware" reading="マルウェア">Malicious Software（悪意のあるソフトウェア）の略語。コンピュータやネットワークに害を与えることを目的としたソフトウェアの総称です。</KeyTerm>には様々な種類があります。

| 種類 | 特徴 |
|---|---|
| ウイルス | 他のプログラムに寄生し、実行時に自己複製する |
| ワーム | 自己複製し、ネットワークを通じて自動的に拡散する |
| トロイの木馬 | 有用なソフトウェアに偽装してシステムに侵入する |
| ランサムウェア | データを暗号化し、復号のために身代金を要求する |
| スパイウェア | ユーザーの行動を秘密裏に監視・記録する |

### DoS / DDoS 攻撃

<KeyTerm term="Denial of Service" reading="サービスきょひこうげき">大量のリクエストやトラフィックを標的のサーバーやネットワークに送り付け、正規ユーザーがサービスを利用できないようにする攻撃です。DoS と略されます。</KeyTerm>攻撃は、CIA トライアドの**可用性**を標的とする攻撃です。

**DDoS（Distributed Denial of Service）** は、多数のコンピュータ（ボットネット）から同時に攻撃を仕掛ける分散型の DoS 攻撃で、防御がより困難です。

<KeyPoint>
- フィッシングはソーシャルエンジニアリング攻撃の一種で、人間の心理を悪用する
- マルウェアにはウイルス、ワーム、トロイの木馬、ランサムウェアなど多くの種類がある
- DoS 攻撃は CIA トライアドの「可用性」を標的とする
- DDoS はボットネットを使った分散型の DoS 攻撃で、より大規模で防御が困難
</KeyPoint>

## まとめ

- CIA トライアド（機密性・完全性・可用性）は情報セキュリティの基本原則であり、すべてのセキュリティ対策はこの3要素のいずれかを保護するためのものである
- AAA フレームワーク（認証・認可・アカウンティング）は、ネットワークリソースへのアクセス制御を体系化する
- MFA は異なるカテゴリの認証要素を組み合わせることで、認証の強度を大幅に向上させる
- 暗号化には対称鍵方式（同じ鍵で暗号化/復号）と非対称鍵方式（公開鍵/秘密鍵ペア）がある
- フィッシング、マルウェア、DoS/DDoS は代表的なセキュリティ脅威であり、それぞれの特徴と対策を理解することが重要
