## 概要

IPv4 アドレスは「ネットワーク部」と「ホスト部」に分かれており、その境界を決めるのがサブネットマスクです。このトピックでは、サブネットマスクの仕組み、CIDR 表記、そしてサブネットの計算方法を学びます。サブネッティングは CCST 試験でも出題頻度が高く、確実に計算できるようにしておく必要があります。

## サブネットマスクとは

<KeyTerm term="Subnet Mask" reading="サブネットマスク">IPv4 アドレスのネットワーク部とホスト部の境界を示す 32 ビットの値。ネットワーク部に対応するビットは 1、ホスト部に対応するビットは 0 で表されます。</KeyTerm>

IPv4 アドレスは「どのネットワークに属するか」を示す **ネットワーク部** と、「そのネットワーク内のどのデバイスか」を示す **ホスト部** の 2 つで構成されます。

例として、IP アドレス `192.168.1.100` にサブネットマスク `255.255.255.0` を適用した場合を見てみましょう。

```
IP アドレス:       192.168.1  .100
サブネットマスク:   255.255.255.0
                   ──────────  ───
                   ネットワーク部  ホスト部
```

サブネットマスクを 2 進数で表すと、連続する 1 のビットがネットワーク部、連続する 0 のビットがホスト部を示します。

```
255.255.255.0 = 11111111.11111111.11111111.00000000
                ├─── ネットワーク部（24ビット）───┤├ホスト部（8ビット）┤
```

### デフォルトサブネットマスク

| クラス | デフォルトサブネットマスク | CIDR | ネットワーク部 | ホスト部 |
|---|---|---|---|---|
| クラス A | 255.0.0.0 | /8 | 8 ビット | 24 ビット |
| クラス B | 255.255.0.0 | /16 | 16 ビット | 16 ビット |
| クラス C | 255.255.255.0 | /24 | 24 ビット | 8 ビット |

<KeyPoint>
- サブネットマスクはネットワーク部（連続する 1）とホスト部（連続する 0）の境界を定義する
- IP アドレスとサブネットマスクの AND 演算でネットワークアドレスが求まる
- サブネットマスクのビットは必ず「連続する 1 のあとに連続する 0」の形式になる
</KeyPoint>

## CIDR 表記

<KeyTerm term="CIDR" reading="サイダー">Classless Inter-Domain Routing の略。IP アドレスのあとにスラッシュとネットワーク部のビット数を付けて表記する方法です。</KeyTerm>

CIDR 表記は、サブネットマスクを簡潔に表す方法です。ネットワーク部のビット数を `/` のあとに記述します。

```
192.168.1.0/24
```

この表記は「IP アドレス 192.168.1.0、サブネットマスク 255.255.255.0（先頭 24 ビットがネットワーク部）」を意味します。

### よく使われるサブネットマスクと CIDR の対応

| CIDR | サブネットマスク | ホスト数 |
|---|---|---|
| /8 | 255.0.0.0 | 16,777,214 |
| /16 | 255.255.0.0 | 65,534 |
| /24 | 255.255.255.0 | 254 |
| /25 | 255.255.255.128 | 126 |
| /26 | 255.255.255.192 | 62 |
| /27 | 255.255.255.224 | 30 |
| /28 | 255.255.255.240 | 14 |
| /29 | 255.255.255.248 | 6 |
| /30 | 255.255.255.252 | 2 |

## サブネッティングの基本計算

<KeyTerm term="Subnetting" reading="サブネッティング">大きなネットワークをより小さなサブネットに分割する技術。効率的な IP アドレス管理とネットワークの整理に使われます。</KeyTerm>

サブネッティングの計算で求められるのは、主に以下の 4 つの値です。

### 1. 使用可能なホスト数の計算

```
使用可能なホスト数 = 2^(ホスト部のビット数) - 2
```

「-2」するのは、**ネットワークアドレス**（ホスト部がすべて 0）と**ブロードキャストアドレス**（ホスト部がすべて 1）を除くためです。

**例:** /24 の場合
- ホスト部 = 32 - 24 = 8 ビット
- 使用可能ホスト数 = 2^8 - 2 = 256 - 2 = **254 台**

### 2. ネットワークアドレスの計算

ネットワークアドレスは、IP アドレスとサブネットマスクの **AND 演算** で求めます。

**例:** IP: 192.168.1.130、サブネットマスク: 255.255.255.192（/26）

```
IP:     192.168.1.  10|000010   (130 = 10000010)
マスク:  255.255.255.11|000000   (192 = 11000000)
AND:    192.168.1.  10|000000   (128 = 10000000)
```

ネットワークアドレス = **192.168.1.128**

### 3. ブロードキャストアドレスの計算

ブロードキャストアドレスは、ネットワークアドレスのホスト部をすべて 1 にした値です。

**例:** ネットワーク 192.168.1.128/26 の場合

```
ネットワーク: 192.168.1.10|000000  (128)
ブロードキャスト: 192.168.1.10|111111  (191)
```

ブロードキャストアドレス = **192.168.1.191**

### 4. 使用可能な IP 範囲

使用可能なホストアドレスの範囲は、ネットワークアドレスの次のアドレスからブロードキャストアドレスの前のアドレスまでです。

**例:** 192.168.1.128/26 の場合
- 使用可能範囲: **192.168.1.129** 〜 **192.168.1.190**
- 使用可能ホスト数: 2^6 - 2 = **62 台**

<KeyPoint>
- 使用可能なホスト数 = 2^(ホスト部ビット数) - 2（ネットワークアドレスとブロードキャストを除く）
- ネットワークアドレスは IP とサブネットマスクの AND 演算で求められる
- ブロードキャストアドレスはホスト部のビットをすべて 1 にした値である
</KeyPoint>

## サブネット計算の実践例

### 例題: 192.168.10.75/28 のサブネット情報を求める

**ステップ 1:** サブネットマスクを確認
- /28 → サブネットマスク = 255.255.255.240
- ホスト部 = 32 - 28 = 4 ビット

**ステップ 2:** ブロックサイズ（サブネットの大きさ）を計算
- ブロックサイズ = 2^4 = 16

**ステップ 3:** ネットワークアドレスを求める
- 75 ÷ 16 = 4 余り 11 → 4 × 16 = 64
- ネットワークアドレス = **192.168.10.64**

**ステップ 4:** ブロードキャストアドレスを求める
- 64 + 16 - 1 = 79
- ブロードキャストアドレス = **192.168.10.79**

**ステップ 5:** 使用可能範囲を求める
- 最初のホスト: **192.168.10.65**
- 最後のホスト: **192.168.10.78**
- 使用可能ホスト数: 2^4 - 2 = **14 台**

### ブロックサイズ法のまとめ

「ブロックサイズ法」は、最後のオクテットのサブネット値（256 - サブネットマスクの最後のオクテット値）を使ってすばやく計算する方法です。

```
ブロックサイズ = 256 - (サブネットマスクの最終オクテット)
```

/28 の場合: 256 - 240 = 16 → サブネットは 0, 16, 32, 48, 64, 80, ... と 16 ごとに区切られます。

<KeyPoint>
- ブロックサイズ法を使うと、暗算でもサブネット計算が可能になる
- ブロックサイズ = 256 - サブネットマスクの最終オクテット値
- 試験では「この IP アドレスが属するサブネットはどれか」という形式で出題されることが多い
</KeyPoint>

## まとめ

- **サブネットマスク** は IP アドレスのネットワーク部とホスト部の境界を定義する 32 ビットの値である
- **CIDR 表記** (/24, /26 など) はサブネットマスクを簡潔に表す方法で、ネットワーク部のビット数を示す
- サブネット計算では **ネットワークアドレス、ブロードキャストアドレス、使用可能ホスト数、使用可能範囲** を求められるようにする
- **ブロックサイズ法** を使うとすばやくサブネット計算ができる（256 - マスクの最終オクテット）
- 使用可能なホスト数は **2^(ホスト部ビット数) - 2** で求められる
