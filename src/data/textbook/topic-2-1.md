## 概要

インターネット上で通信を行うには IP アドレスが必要ですが、すべてのデバイスにグローバルな IP アドレスを割り当てるのは現実的ではありません。そこで、組織内部で使う「プライベート IP アドレス」と、インターネット上で使う「パブリック IP アドレス」が区別されています。このトピックでは、IPv4 アドレスのクラス分類、プライベートアドレスの範囲、そして NAT による変換の仕組みを学びます。

## IPv4 アドレスの基本

IPv4 アドレスは **32 ビット** の数値で、8 ビットずつ 4 つのグループ（オクテット）に分け、各オクテットを 10 進数で表記します。

```
例: 192.168.1.100
```

各オクテットは 0〜255 の範囲の値を取ります。32 ビットで表現できるアドレスの総数は約 43 億（2^32 = 4,294,967,296）個です。

## IPv4 アドレスのクラス

<KeyTerm term="Address Class" reading="アドレスクラス">IPv4 アドレスを先頭ビットのパターンに基づいて分類する方法。クラス A〜E の 5 つに分かれます。</KeyTerm>

IPv4 アドレスは歴史的に、先頭のビットパターンに基づいて 5 つのクラスに分類されてきました。

### クラス A

- **範囲**: 1.0.0.0 〜 126.255.255.255
- **先頭ビット**: 0（第 1 オクテットが 1〜126）
- **デフォルトサブネットマスク**: 255.0.0.0（/8）
- **ネットワーク部**: 最初の 8 ビット、**ホスト部**: 残りの 24 ビット
- **特徴**: 少数の大規模ネットワーク向け。1 ネットワークあたり約 1,677 万台のホストを収容可能

### クラス B

- **範囲**: 128.0.0.0 〜 191.255.255.255
- **先頭ビット**: 10（第 1 オクテットが 128〜191）
- **デフォルトサブネットマスク**: 255.255.0.0（/16）
- **ネットワーク部**: 最初の 16 ビット、**ホスト部**: 残りの 16 ビット
- **特徴**: 中規模ネットワーク向け。1 ネットワークあたり約 65,534 台のホストを収容可能

### クラス C

- **範囲**: 192.0.0.0 〜 223.255.255.255
- **先頭ビット**: 110（第 1 オクテットが 192〜223）
- **デフォルトサブネットマスク**: 255.255.255.0（/24）
- **ネットワーク部**: 最初の 24 ビット、**ホスト部**: 残りの 8 ビット
- **特徴**: 多数の小規模ネットワーク向け。1 ネットワークあたり 254 台のホストを収容可能

### クラス D

- **範囲**: 224.0.0.0 〜 239.255.255.255
- **用途**: **マルチキャスト** 専用。1 つのパケットを特定のグループに同時送信するために使用

### クラス E

- **範囲**: 240.0.0.0 〜 255.255.255.255
- **用途**: **実験用** として予約されており、一般的には使用されない

<KeyPoint>
- クラス A（1〜126）は大規模ネットワーク、クラス B（128〜191）は中規模、クラス C（192〜223）は小規模向け
- クラス D（224〜239）はマルチキャスト専用、クラス E（240〜255）は実験用
- 127.x.x.x はループバックアドレス（自分自身を示す特殊アドレス）として予約されている
</KeyPoint>

## 特殊な IP アドレス

いくつかの IP アドレスは特別な目的に予約されています。

| アドレス | 用途 |
|---|---|
| 0.0.0.0 | 「すべてのネットワーク」を示す（デフォルトルートなどで使用） |
| 127.0.0.1 | ループバックアドレス（自分自身への通信テスト用） |
| 255.255.255.255 | リミテッドブロードキャスト（同一ネットワーク内の全デバイスへ送信） |

<KeyTerm term="Loopback Address" reading="ループバックアドレス">127.0.0.0/8 の範囲に属するアドレス。自分自身への通信に使用され、ネットワークインターフェースの動作確認に利用されます。</KeyTerm>

## プライベート IP アドレス

<KeyTerm term="Private IP Address" reading="プライベートアイピーアドレス">組織内のネットワークでのみ使用される IP アドレス。インターネット上ではルーティングされず、自由に使用できます。</KeyTerm>

RFC 1918 で定められたプライベート IP アドレスの範囲は以下の 3 つです。

| クラス | プライベートアドレス範囲 | CIDR 表記 | アドレス数 |
|---|---|---|---|
| クラス A | 10.0.0.0 〜 10.255.255.255 | 10.0.0.0/8 | 約 1,677 万 |
| クラス B | 172.16.0.0 〜 172.31.255.255 | 172.16.0.0/12 | 約 104 万 |
| クラス C | 192.168.0.0 〜 192.168.255.255 | 192.168.0.0/16 | 約 65,534 |

プライベート IP アドレスは、家庭やオフィスなどの内部ネットワークで使用されます。異なる組織が同じプライベートアドレスを使用しても、インターネット上では衝突しません（インターネットにはルーティングされないため）。

家庭用ルーターでは通常 192.168.x.x の範囲が使われ、大企業では 10.x.x.x の範囲が使われることが多いです。

## パブリック IP アドレス

<KeyTerm term="Public IP Address" reading="パブリックアイピーアドレス">インターネット上で一意に割り当てられる IP アドレス。世界中で重複せず、インターネット上でルーティング可能です。</KeyTerm>

パブリック IP アドレスは、プライベートアドレス範囲および特殊アドレスを除いた、インターネット上で使用可能なアドレスです。IANA（Internet Assigned Numbers Authority）が管理し、各地域のレジストリ（APNIC、ARIN など）を通じて ISP や組織に割り当てられます。

パブリック IP アドレスは有限であり、IPv4 アドレスの枯渇が深刻な問題となっています。この問題に対処するために、プライベートアドレスと NAT の組み合わせ、そして IPv6 への移行が進められています。

<KeyPoint>
- プライベートアドレスの 3 つの範囲（10.x、172.16〜31.x、192.168.x）は必ず暗記する
- プライベートアドレスはインターネット上でルーティングされない（組織内部でのみ有効）
- パブリックアドレスはインターネット上で一意であり、枯渇が問題になっている
</KeyPoint>

## NAT（Network Address Translation）

<KeyTerm term="NAT" reading="ナット">プライベート IP アドレスをパブリック IP アドレスに変換する技術。組織内のデバイスが少数のパブリックアドレスでインターネットにアクセスできるようにします。</KeyTerm>

NAT は、プライベートアドレスを使う内部ネットワークのデバイスがインターネットと通信する際に、ルーター上でアドレスの変換を行います。

### スタティック NAT

1 つのプライベートアドレスを 1 つの固定パブリックアドレスに変換します。サーバーの公開など、常に同じパブリックアドレスが必要な場合に使用します。

### ダイナミック NAT

複数のプライベートアドレスを、プールされた複数のパブリックアドレスに動的に割り当てます。

### PAT（Port Address Translation）

<KeyTerm term="PAT" reading="パット">NAT の一種で、1 つのパブリック IP アドレスをポート番号で区別して、複数のプライベートアドレスと共有する技術。NAT オーバーロードとも呼ばれます。</KeyTerm>

PAT は最も一般的な NAT の形態で、家庭用ルーターでも広く使われています。1 つのパブリック IP アドレスを、送信元ポート番号を変えることで複数の内部デバイスが共有します。

**PAT の動作例:**
- PC-A（192.168.1.10:50001）→ ルーター → インターネット（203.0.113.1:60001）
- PC-B（192.168.1.20:50002）→ ルーター → インターネット（203.0.113.1:60002）

どちらも同じパブリック IP（203.0.113.1）を使いますが、ポート番号が異なるため区別できます。

<KeyPoint>
- NAT はプライベートアドレスをパブリックアドレスに変換する技術で、IPv4 アドレス枯渇対策の一つ
- PAT（NAT オーバーロード）は 1 つのパブリック IP を複数のデバイスで共有する最も一般的な方式
- 家庭用ルーターはほぼすべて PAT を使用している
</KeyPoint>

## まとめ

- IPv4 アドレスは **クラス A〜E** に分類され、クラス A〜C が一般的な通信に使用される
- **プライベート IP アドレス**（10.x、172.16〜31.x、192.168.x）は組織内部でのみ使用され、インターネット上ではルーティングされない
- **パブリック IP アドレス** はインターネット上で一意であり、枯渇が進んでいる
- **NAT** はプライベートアドレスとパブリックアドレスを変換する技術で、特に **PAT** が広く普及している
- 試験ではプライベートアドレスの範囲、クラスの判別、NAT の役割が頻出する
