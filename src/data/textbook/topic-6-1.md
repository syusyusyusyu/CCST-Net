## 概要

ファイアウォールは、ネットワークセキュリティの最も基本的かつ重要な防御手段です。信頼できるネットワーク（内部）と信頼できないネットワーク（外部、主にインターネット）の間に配置され、通過するトラフィックを検査して許可または拒否の判断を行います。このトピックでは、ファイアウォールの動作原理、ブロックすべきポートとプロトコル、許可/拒否ルールの仕組み、そしてネットワーク内での適切な配置場所について学びます。

## ファイアウォールとは

<KeyTerm term="Firewall" reading="ファイアウォール">ネットワークトラフィックを監視し、事前に定義されたセキュリティルールに基づいて通信の許可または拒否を行うネットワークセキュリティデバイスまたはソフトウェアです。</KeyTerm>

ファイアウォールの名前は、建築用語の「防火壁」に由来しています。建物の防火壁が火災の拡大を防ぐように、ネットワークのファイアウォールは不正なトラフィックの侵入を防ぎます。

ファイアウォールには大きく分けて2つの種類があります。

**ハードウェアファイアウォール**は、専用のネットワーク機器として動作します。企業ネットワークの境界に設置され、高いスループットで大量のトラフィックを処理できます。Cisco ASA（Adaptive Security Appliance）や Cisco Firepower はその代表例です。

**ソフトウェアファイアウォール**は、コンピュータ上で動作するプログラムです。Windows Defender ファイアウォールや Linux の iptables がこれに該当します。個々のホストを保護するため、<KeyTerm term="Host-based Firewall" reading="ホストベースファイアウォール">個々のコンピュータにインストールされ、そのコンピュータへの通信を制御するファイアウォールです。</KeyTerm>とも呼ばれます。

<KeyPoint>
- ファイアウォールにはハードウェア型とソフトウェア型がある
- ハードウェア型はネットワーク境界に配置し、ソフトウェア型は個々のホストで動作する
- 企業ネットワークでは両方を組み合わせた多層防御が一般的
</KeyPoint>

## ポートとプロトコルの制御

ファイアウォールの主要な機能の1つは、<KeyTerm term="Port" reading="ポート">TCP/IP 通信において、特定のアプリケーションやサービスを識別するための番号（0〜65535）です。</KeyTerm>番号に基づいてトラフィックをフィルタリングすることです。

### よく使用されるポート番号

ネットワーク管理者は、以下のポート番号を把握しておく必要があります。

| ポート番号 | プロトコル | 用途 | 一般的な方針 |
|---|---|---|---|
| 20, 21 | FTP | ファイル転送 | 外部からはブロック推奨 |
| 22 | SSH | セキュアなリモートアクセス | 必要に応じて許可 |
| 23 | Telnet | リモートアクセス（非暗号化） | ブロック推奨 |
| 25 | SMTP | メール送信 | メールサーバーのみ許可 |
| 53 | DNS | 名前解決 | DNS サーバーのみ許可 |
| 80 | HTTP | Web（非暗号化） | 通常許可 |
| 443 | HTTPS | Web（暗号化） | 通常許可 |
| 3389 | RDP | リモートデスクトップ | 外部からはブロック推奨 |

### ブロックすべきポートとプロトコル

セキュリティの観点から、以下のようなトラフィックは原則としてブロックすべきです。

- **Telnet（ポート23）**: 通信が暗号化されないため、代わりに SSH（ポート22）を使用します
- **未使用のポート**: 使用していないポートはすべて閉じるのが基本原則です
- **ICMP の一部**: ping フラッド攻撃を防ぐため、外部からの ICMP を制限する場合があります

<KeyPoint>
- ファイアウォールはポート番号を基にトラフィックを制御する
- Telnet（ポート23）は暗号化されないため、SSH（ポート22）を使用すべき
- HTTP（80）と HTTPS（443）は一般的に許可し、不要なポートはすべてブロックする
- CCST 試験では主要なポート番号とプロトコルの対応を覚えることが重要
</KeyPoint>

## 許可/拒否ルール（ACL）

ファイアウォールは<KeyTerm term="Access Control List" reading="アクセスコントロールリスト">ネットワークトラフィックに対する許可（permit）と拒否（deny）の条件を定義したルールのリストです。ACL と略されます。</KeyTerm>を使用してトラフィックを制御します。

### ルールの構成要素

ACL の各ルール（エントリ）は、通常以下の要素で構成されます。

1. **アクション**: 許可（permit / allow）または 拒否（deny / block）
2. **送信元 IP アドレス**: トラフィックの発信元
3. **宛先 IP アドレス**: トラフィックの送信先
4. **プロトコル**: TCP、UDP、ICMP など
5. **ポート番号**: 宛先ポート（またはポート範囲）

### ルールの評価順序

ACL のルールは**上から順に評価**されます。トラフィックが最初に一致したルールが適用され、それ以降のルールは評価されません。この仕組みを理解することは非常に重要です。

例えば、以下のようなルールセットを考えます。

```
ルール1: PERMIT TCP 任意 → 192.168.1.10 ポート443
ルール2: DENY TCP 任意 → 192.168.1.10 任意ポート
ルール3: PERMIT TCP 任意 → 任意 ポート80
```

この場合、192.168.1.10 への HTTPS（443）通信はルール1で許可されますが、同じサーバーへの他のTCP通信はルール2で拒否されます。

### 暗黙の拒否

多くのファイアウォールには、ルールリストの最後に<KeyTerm term="Implicit Deny" reading="あんもくのきょひ">ACL の末尾に自動的に存在する、すべてのトラフィックを拒否するルールです。明示的なルールに一致しない通信はすべてブロックされます。</KeyTerm>が存在します。つまり、明示的に許可されていない通信はすべてブロックされます。これは「ホワイトリスト方式」とも呼ばれ、セキュリティの基本原則である**最小権限の原則**に基づいています。

<KeyPoint>
- ACL ルールは上から順に評価され、最初に一致したルールが適用される
- ルールの順序が結果に大きく影響するため、設計時は順番に注意が必要
- 暗黙の拒否（Implicit Deny）により、許可ルールに一致しないトラフィックはすべてブロックされる
</KeyPoint>

## ファイアウォールの配置場所

ファイアウォールの配置場所は、保護対象と脅威モデルによって決まります。

### ネットワーク境界（ペリメータ）

最も一般的な配置は、内部ネットワークとインターネットの間です。すべての外部トラフィックがファイアウォールを通過するため、不正アクセスの最初の防御線となります。

### DMZ（非武装地帯）

<KeyTerm term="DMZ" reading="ディーエムゼット">Demilitarized Zone の略。外部に公開するサーバー（Web サーバー、メールサーバーなど）を配置する、内部ネットワークとインターネットの間に設けられた中間的なネットワークセグメントです。</KeyTerm>は、公開サーバーを配置するための特別なネットワークゾーンです。

DMZ の構成では、通常2つのファイアウォール（または1台のファイアウォールに3つのインターフェース）を使用します。

```
インターネット → [外部FW] → DMZ (Webサーバー等) → [内部FW] → 内部ネットワーク
```

- **外部ファイアウォール**: インターネットからDMZへの必要な通信のみ許可
- **内部ファイアウォール**: DMZから内部ネットワークへのアクセスを厳しく制限

この構成により、仮に DMZ のサーバーが攻撃を受けても、内部ネットワークは内部ファイアウォールによって保護されます。

### ネットワークセグメント間

大規模な企業ネットワークでは、部門間やフロア間にもファイアウォールを配置して、内部のセグメント間トラフィックを制御することがあります。これを<KeyTerm term="Internal Firewall" reading="ないぶファイアウォール">組織の内部ネットワーク内に配置され、異なるネットワークセグメント間のトラフィックを制御するファイアウォールです。</KeyTerm>と呼びます。

<KeyPoint>
- ファイアウォールは通常、内部ネットワークとインターネットの境界に配置する
- DMZ を使うことで、公開サーバーを内部ネットワークから分離できる
- 多層防御の考え方に基づき、ネットワーク境界・DMZ・ホストの各レベルでファイアウォールを配置する
</KeyPoint>

## ステートフルインスペクションとパケットフィルタリング

ファイアウォールの検査方式には主に2つの種類があります。

**パケットフィルタリング**は、個々のパケットのヘッダー情報（送信元/宛先 IP、ポート番号、プロトコル）のみを検査します。シンプルで高速ですが、通信の文脈（コンテキスト）を理解できません。

<KeyTerm term="Stateful Inspection" reading="ステートフルインスペクション">通信のセッション状態（接続の確立、データ転送、切断）を追跡し、正当な通信の一部であるかどうかを判断するファイアウォールの検査方式です。</KeyTerm>は、通信セッション全体の状態を追跡します。例えば、内部から開始された HTTP 接続の応答パケットは自動的に許可されますが、外部から突然送られてきた応答パケットはブロックされます。現在のほとんどのファイアウォールはこの方式を採用しています。

## まとめ

- ファイアウォールはネットワークトラフィックを監視し、ルールに基づいて許可/拒否を行うセキュリティ機構である
- ポート番号（HTTP: 80、HTTPS: 443、SSH: 22、Telnet: 23 など）とプロトコルの対応を覚えることが試験対策として重要
- ACL ルールは上から順に評価され、暗黙の拒否（Implicit Deny）が末尾に存在する
- ファイアウォールはネットワーク境界、DMZ、ホストの各レベルに配置して多層防御を実現する
- ステートフルインスペクションは通信の状態を追跡し、より高度なフィルタリングを行う
