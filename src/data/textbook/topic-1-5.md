## 概要

ネットワーク上でデバイス同士が通信するためには、共通のルール（プロトコル）が必要です。このトピックでは、CCST 試験で出題される主要なネットワークプロトコルについて、その役割・動作・使用するポート番号を学びます。TCP と UDP の違いを理解し、各プロトコルがどちらを使うかを把握することが重要です。

## TCP と UDP

### TCP（Transmission Control Protocol）

<KeyTerm term="TCP" reading="ティーシーピー">コネクション型の信頼性のある通信プロトコル。データの到達確認や順序保証を行い、確実なデータ転送を実現します。</KeyTerm>

TCP はデータを送信する前に、送信側と受信側の間で接続を確立します。この接続確立プロセスを **3 ウェイハンドシェイク** と呼びます。

**3 ウェイハンドシェイクの流れ:**
1. **SYN**: クライアントがサーバーに接続要求を送信
2. **SYN-ACK**: サーバーが要求を受け入れ、応答を返送
3. **ACK**: クライアントが確認応答を送信 → 接続確立

TCP の特徴:
- データの到達を確認（ACK による確認応答）
- パケットの順序を保証（シーケンス番号を使用）
- エラー検出と再送制御
- フロー制御（受信側の処理能力に合わせて送信速度を調整）

### UDP（User Datagram Protocol）

<KeyTerm term="UDP" reading="ユーディーピー">コネクションレス型の軽量な通信プロトコル。到達確認を行わないため高速だが、信頼性は保証されません。</KeyTerm>

UDP は接続の確立や到達確認を行いません。データを送り出すだけで、届いたかどうかは確認しません。そのため TCP より高速で、オーバーヘッドが少ないのが特徴です。

### TCP と UDP の比較

| 特徴 | TCP | UDP |
|---|---|---|
| 接続方式 | コネクション型 | コネクションレス型 |
| 信頼性 | 高い（到達確認あり） | 低い（到達確認なし） |
| 順序保証 | あり | なし |
| 速度 | 比較的遅い | 高速 |
| オーバーヘッド | 大きい | 小さい |
| 用途 | Web、メール、ファイル転送 | 動画配信、VoIP、DNS |

<KeyPoint>
- TCP は信頼性重視の通信に使われ、3 ウェイハンドシェイクで接続を確立する
- UDP は速度重視の通信に使われ、接続確立や到達確認を省略する
- 試験では「このプロトコルは TCP と UDP のどちらを使うか」がよく問われる
</KeyPoint>

## Web 関連プロトコル

### HTTP（HyperText Transfer Protocol）

<KeyTerm term="HTTP" reading="エイチティーティーピー">Web ページの転送に使用されるプロトコル。TCP ポート 80 を使用します。通信内容は暗号化されません。</KeyTerm>

HTTP はクライアント（ブラウザ）とサーバーの間で Web コンテンツをやり取りするプロトコルです。GET（データ取得）や POST（データ送信）などのメソッドを使用します。

### HTTPS（HTTP Secure）

<KeyTerm term="HTTPS" reading="エイチティーティーピーエス">HTTP に TLS/SSL による暗号化を加えたプロトコル。TCP ポート 443 を使用します。</KeyTerm>

HTTPS は HTTP の通信を暗号化し、データの盗聴や改ざんを防ぎます。現在の Web サイトの大半は HTTPS を使用しています。URL が `https://` で始まり、ブラウザに鍵アイコンが表示されます。

## ファイル転送プロトコル

### FTP（File Transfer Protocol）

<KeyTerm term="FTP" reading="エフティーピー">ファイル転送に使用されるプロトコル。TCP ポート 20（データ転送）と 21（制御）を使用します。通信は暗号化されません。</KeyTerm>

FTP はサーバーとクライアント間でファイルをアップロード・ダウンロードするための古くからあるプロトコルです。ユーザー名とパスワードによる認証を行いますが、通信内容（パスワードを含む）が平文で送信されるため、セキュリティ上のリスクがあります。

### SFTP（SSH File Transfer Protocol）

<KeyTerm term="SFTP" reading="エスエフティーピー">SSH の暗号化技術を利用した安全なファイル転送プロトコル。TCP ポート 22 を使用します。</KeyTerm>

SFTP は FTP と同様のファイル転送機能を持ちますが、SSH（Secure Shell）の暗号化チャネル上で動作するため、通信内容が暗号化されます。セキュリティが求められる環境では FTP の代わりに SFTP が推奨されます。

### TFTP（Trivial File Transfer Protocol）

<KeyTerm term="TFTP" reading="ティーエフティーピー">認証機能を持たない簡易的なファイル転送プロトコル。UDP ポート 69 を使用します。</KeyTerm>

TFTP は非常にシンプルなファイル転送プロトコルで、認証機能がありません。主にネットワーク機器のファームウェア更新や、ディスクレスワークステーションの起動イメージの転送に使われます。UDP を使用するため軽量ですが、信頼性は低くなります。

<KeyPoint>
- FTP はポート 20/21（TCP）で暗号化なし、SFTP はポート 22（TCP）で暗号化あり
- TFTP はポート 69（UDP）で認証なし、ネットワーク機器の設定転送に使われる
- セキュアなファイル転送には SFTP が推奨される
</KeyPoint>

## ネットワーク基盤プロトコル

### DHCP（Dynamic Host Configuration Protocol）

<KeyTerm term="DHCP" reading="ディーエイチシーピー">ネットワーク上のデバイスに IP アドレスなどのネットワーク設定を自動的に割り当てるプロトコル。UDP ポート 67（サーバー）/ 68（クライアント）を使用します。</KeyTerm>

DHCP を使うと、デバイスがネットワークに接続した際に、IP アドレス、サブネットマスク、デフォルトゲートウェイ、DNS サーバーのアドレスなどが自動的に割り当てられます。

**DHCP の動作（DORA プロセス）:**
1. **Discover**: クライアントがネットワーク上の DHCP サーバーを探す（ブロードキャスト）
2. **Offer**: DHCP サーバーが IP アドレスを提案
3. **Request**: クライアントが提案された IP アドレスの使用を要求
4. **Acknowledge**: サーバーが割り当てを確認し、リース期間を通知

### DNS（Domain Name System）

<KeyTerm term="DNS" reading="ディーエヌエス">ドメイン名（例: www.cisco.com）を IP アドレス（例: 72.163.4.185）に変換する名前解決システム。UDP/TCP ポート 53 を使用します。</KeyTerm>

DNS はインターネットの「電話帳」とも呼ばれます。人間が覚えやすいドメイン名を、コンピュータが理解できる IP アドレスに変換します。通常の問い合わせには UDP ポート 53 を使用しますが、512 バイトを超える応答やゾーン転送には TCP ポート 53 を使用します。

### ICMP（Internet Control Message Protocol）

<KeyTerm term="ICMP" reading="アイシーエムピー">ネットワークの診断やエラー通知に使用されるプロトコル。ping コマンドや traceroute で利用されます。</KeyTerm>

ICMP は IP の補助プロトコルで、ネットワークの到達性確認（ping）や経路追跡（traceroute）に使用されます。ICMP はトランスポート層のプロトコルではなく、ネットワーク層（IP と同じ層）で動作します。ポート番号は使用しません。

### NTP（Network Time Protocol）

<KeyTerm term="NTP" reading="エヌティーピー">ネットワーク上のデバイスの時刻を同期するプロトコル。UDP ポート 123 を使用します。</KeyTerm>

NTP はネットワーク上のすべてのデバイスの時計を正確に合わせるために使用されます。ログの時刻を統一するため、セキュリティ監査やトラブルシューティングにおいて正確な時刻同期は非常に重要です。

## 主要プロトコルのポート番号一覧

| プロトコル | ポート番号 | TCP/UDP | 用途 |
|---|---|---|---|
| FTP (データ) | 20 | TCP | ファイル転送（データ） |
| FTP (制御) | 21 | TCP | ファイル転送（制御） |
| SSH / SFTP | 22 | TCP | 安全なリモートアクセス / ファイル転送 |
| Telnet | 23 | TCP | リモートアクセス（暗号化なし） |
| DNS | 53 | UDP / TCP | 名前解決 |
| DHCP | 67 / 68 | UDP | IP アドレスの自動割り当て |
| TFTP | 69 | UDP | 簡易ファイル転送 |
| HTTP | 80 | TCP | Web ページ転送 |
| NTP | 123 | UDP | 時刻同期 |
| HTTPS | 443 | TCP | 暗号化された Web ページ転送 |

<KeyPoint>
- DHCP は DORA プロセス（Discover → Offer → Request → Acknowledge）で IP アドレスを自動割り当てする
- DNS はドメイン名を IP アドレスに変換する仕組みで、通常 UDP ポート 53 を使用する
- 主要プロトコルのポート番号（特に 20/21, 22, 23, 53, 67/68, 69, 80, 123, 443）は暗記が必須
</KeyPoint>

## まとめ

- **TCP** は信頼性重視、**UDP** は速度重視のトランスポート層プロトコルである
- **HTTP**（80）と **HTTPS**（443）は Web 通信、**FTP**（20/21）・**SFTP**（22）・**TFTP**（69）はファイル転送に使用される
- **DHCP**（67/68）は IP アドレスの自動割り当て、**DNS**（53）はドメイン名から IP アドレスへの変換を行う
- **ICMP** は ping や traceroute で使われるネットワーク診断プロトコルで、ポート番号を使用しない
- 各プロトコルの役割、使用するポート番号、TCP/UDP の区別は CCST 試験の頻出範囲である
