## 概要

IT インフラの運用形態として、従来型の「オンプレミス」と、インターネット経由でリソースを利用する「クラウド」があります。このトピックでは、オンプレミスとクラウドの違い、クラウドの 3 つの展開モデル（パブリック・プライベート・ハイブリッド）、そしてクラウドの 3 つのサービスモデル（SaaS・PaaS・IaaS）について学びます。

## オンプレミスとは

<KeyTerm term="On-Premises" reading="オンプレミス">自社の施設内に IT 機器（サーバー、ネットワーク機器、ストレージなど）を設置し、自ら管理・運用する形態です。</KeyTerm>

オンプレミスでは、サーバーやネットワーク機器を自社のサーバールームやデータセンターに設置します。ハードウェアの購入、設置、管理、保守をすべて自社で行います。

**オンプレミスの利点:**
- データを自社施設内で完全に管理できるため、セキュリティポリシーを自由に設定可能
- カスタマイズの自由度が高い
- インターネット接続が不要（閉域ネットワークで運用可能）

**オンプレミスの課題:**
- 初期投資（CapEx：設備投資）が大きい
- ハードウェアの調達・設置に時間がかかる
- 機器の保守・更新を自社で行う必要がある
- 急なアクセス増加への対応が難しい（スケーラビリティの制約）

## クラウドコンピューティングとは

<KeyTerm term="Cloud Computing" reading="クラウドコンピューティング">インターネットを通じて、サーバー・ストレージ・ネットワーク・ソフトウェアなどの IT リソースを必要な分だけ利用できるサービスモデルです。</KeyTerm>

クラウドでは、IT リソースをクラウドプロバイダー（AWS、Azure、Google Cloud など）が提供するデータセンターから利用します。自社で機器を購入・管理する必要がなく、使った分だけ支払う従量課金モデルが一般的です。

**クラウドの利点:**
- 初期投資が少なく、運用コスト（OpEx：運用経費）として処理できる
- 必要に応じてリソースを増減できる（スケーラビリティ）
- 導入が迅速（数分でサーバーを構築可能）
- プロバイダーがハードウェアの保守・セキュリティパッチを管理

**クラウドの課題:**
- インターネット接続が必要
- データが外部のデータセンターに保管されるため、規制やコンプライアンスへの対応が必要
- 長期的にはオンプレミスよりコストが高くなる場合がある
- プロバイダーへの依存（ベンダーロックイン）のリスク

<KeyPoint>
- オンプレミスは自社で機器を所有・管理する形態であり、初期投資が大きいが管理の自由度が高い
- クラウドはインターネット経由でリソースを利用する形態であり、初期投資が少なくスケーラビリティに優れる
- オンプレミスのコストは CapEx（設備投資）、クラウドのコストは OpEx（運用経費）として分類される
</KeyPoint>

## クラウドの展開モデル

クラウドの展開モデルは、誰がクラウドリソースを利用できるかによって 3 種類に分類されます。

### パブリッククラウド（Public Cloud）

<KeyTerm term="Public Cloud" reading="パブリッククラウド">クラウドプロバイダーが所有・運営するインフラを、インターネット経由で不特定多数のユーザー（組織）に提供するモデルです。</KeyTerm>

パブリッククラウドでは、複数の顧客（テナント）が同じ物理インフラを共有します。代表的なサービスとして AWS（Amazon Web Services）、Microsoft Azure、Google Cloud Platform があります。

- **利点**: 初期投資不要、従量課金、高いスケーラビリティ
- **課題**: 他のテナントとリソースを共有するため、セキュリティやパフォーマンスに懸念がある場合がある

### プライベートクラウド（Private Cloud）

<KeyTerm term="Private Cloud" reading="プライベートクラウド">特定の組織専用に構築・運用されるクラウド環境。自社データセンターまたは専用のホスティング施設に設置されます。</KeyTerm>

プライベートクラウドでは、インフラを 1 つの組織が専有します。オンプレミスの管理性とクラウドの柔軟性を両立させた形態です。

- **利点**: 高いセキュリティとプライバシー、カスタマイズの自由度
- **課題**: 構築・運用コストが高い、自社での管理負担がある

### ハイブリッドクラウド（Hybrid Cloud）

<KeyTerm term="Hybrid Cloud" reading="ハイブリッドクラウド">パブリッククラウドとプライベートクラウド（またはオンプレミス）を組み合わせて利用するモデルです。</KeyTerm>

ハイブリッドクラウドでは、機密データはプライベート環境に置き、Web サイトやアプリケーションのフロントエンドはパブリッククラウドに置くといった使い分けが可能です。

- **利点**: 用途に応じた最適な環境の選択、段階的なクラウド移行が可能
- **課題**: 複数環境の管理が複雑、環境間の接続設計が必要

<KeyPoint>
- パブリッククラウドは不特定多数が共有するインフラで、コスト効率とスケーラビリティに優れる
- プライベートクラウドは特定組織専用の環境で、セキュリティとカスタマイズ性が高い
- ハイブリッドクラウドは両者を組み合わせた柔軟なモデルであり、多くの企業が採用している
</KeyPoint>

## クラウドのサービスモデル

クラウドサービスは、プロバイダーが管理する範囲と利用者が管理する範囲によって 3 つのモデルに分類されます。

### IaaS（Infrastructure as a Service）

<KeyTerm term="IaaS" reading="アイアース">仮想マシン、ストレージ、ネットワークなどのインフラをクラウドから提供するサービスモデル。OS やミドルウェア以上は利用者が管理します。</KeyTerm>

IaaS では、サーバーの CPU、メモリ、ストレージ、ネットワークといった基盤部分をクラウドプロバイダーが提供します。利用者は OS のインストール、アプリケーションの構築、セキュリティ設定を自分で行います。

**例:** AWS EC2、Azure Virtual Machines、Google Compute Engine

### PaaS（Platform as a Service）

<KeyTerm term="PaaS" reading="パース">アプリケーションの開発・実行環境をクラウドから提供するサービスモデル。OS やミドルウェアの管理はプロバイダーが担当します。</KeyTerm>

PaaS では、IaaS の機能に加えて、OS、ミドルウェア、データベース、開発ツールなどのプラットフォームが提供されます。利用者はアプリケーションのコードとデータだけを管理すればよく、インフラの管理から解放されます。

**例:** AWS Elastic Beanstalk、Azure App Service、Google App Engine

### SaaS（Software as a Service）

<KeyTerm term="SaaS" reading="サース">完成されたソフトウェアをインターネット経由で提供するサービスモデル。利用者はブラウザなどからすぐに利用できます。</KeyTerm>

SaaS では、アプリケーションそのものがクラウドサービスとして提供されます。利用者はソフトウェアのインストールやインフラの管理を一切行う必要がありません。

**例:** Microsoft 365、Google Workspace、Salesforce、Zoom

### サービスモデルの管理範囲の比較

| 管理項目 | オンプレミス | IaaS | PaaS | SaaS |
|---|---|---|---|---|
| アプリケーション | 利用者 | 利用者 | 利用者 | **プロバイダー** |
| データ | 利用者 | 利用者 | 利用者 | **プロバイダー** |
| ミドルウェア | 利用者 | 利用者 | **プロバイダー** | **プロバイダー** |
| OS | 利用者 | 利用者 | **プロバイダー** | **プロバイダー** |
| 仮想化基盤 | 利用者 | **プロバイダー** | **プロバイダー** | **プロバイダー** |
| サーバー・ストレージ | 利用者 | **プロバイダー** | **プロバイダー** | **プロバイダー** |
| ネットワーク | 利用者 | **プロバイダー** | **プロバイダー** | **プロバイダー** |

<KeyPoint>
- IaaS はインフラのみ提供（OS 以上は利用者管理）、自由度が最も高い
- PaaS は開発プラットフォームまで提供（アプリとデータのみ利用者管理）
- SaaS は完成したソフトウェアを提供（利用者は使うだけ）
- 「IaaS → PaaS → SaaS」の順にプロバイダーの管理範囲が広がり、利用者の管理負担が減る
</KeyPoint>

## まとめ

- **オンプレミス** は自社施設に機器を設置して管理する形態で、管理の自由度が高いが初期投資が大きい
- **クラウド** はインターネット経由で IT リソースを利用する形態で、スケーラビリティとコスト効率に優れる
- クラウドの展開モデルには **パブリック**・**プライベート**・**ハイブリッド** の 3 種類がある
- クラウドのサービスモデルには **IaaS**・**PaaS**・**SaaS** の 3 種類があり、プロバイダーの管理範囲が異なる
- 試験では各モデルの特徴と具体例、管理範囲の違いを正確に理解しておくことが重要である
