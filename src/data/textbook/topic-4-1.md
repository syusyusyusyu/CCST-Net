## 概要

Cisco のネットワークデバイス（スイッチやルーター）には、デバイスの状態を視覚的に示す LED インジケーターが搭載されています。これらの LED の色や点滅パターンを読み取ることで、デバイスが正常に動作しているか、エラーが発生しているかを素早く判断できます。CCST 試験では、LED の色と状態の意味について問われることがあります。

## LED インジケーターの基本

Cisco デバイスの LED は、主に以下の 3 つの色で状態を示します。

<KeyTerm term="LED Indicator" reading="エルイーディーインジケーター">Light Emitting Diode（発光ダイオード）を使用した状態表示灯です。ネットワークデバイスの前面パネルに配置されており、デバイスやポートの動作状態を色や点滅パターンで示します。</KeyTerm>

| LED の色 | 一般的な意味 |
|---|---|
| 緑（Green） | 正常動作中 |
| 橙/アンバー（Amber） | 警告・エラー・初期化中 |
| 消灯（Off） | 電源オフ・未接続・無効 |

LED が点灯（常時点灯）している場合と点滅している場合では意味が異なります。点灯は安定した状態を示し、点滅はデータの送受信やアクティビティがあることを示します。

<KeyPoint>
- 緑の点灯 = 正常動作、緑の点滅 = データ送受信中
- アンバーの点灯 = エラーまたは警告状態
- 消灯 = 電源オフ、リンクなし、またはポート無効
</KeyPoint>

## システム LED

Cisco スイッチの前面パネルには、デバイス全体の状態を示すシステム LED があります。

<KeyTerm term="System LED" reading="システムエルイーディー">デバイス全体の電源状態や動作状態を示す LED です。通常、デバイスの前面パネル左側に配置されています。</KeyTerm>

**システム LED の状態:**

| 状態 | 意味 |
|---|---|
| 消灯 | システムの電源が入っていません |
| 緑 | システムは正常に動作しています |
| アンバー | システムは電源が入っていますが、正常に動作していません（POST エラーなど） |

POST（Power-On Self-Test）は、デバイスの電源投入時に実行される自己診断テストです。POST に失敗するとシステム LED がアンバーになり、デバイスに何らかのハードウェア問題があることを示します。

## ポート LED

各イーサネットポートにも LED が搭載されており、ポートごとの接続状態を確認できます。

<KeyTerm term="Port LED" reading="ポートエルイーディー">各物理ポートに隣接して配置された LED で、そのポートのリンク状態、速度、アクティビティなどを示します。</KeyTerm>

**ポート LED の状態:**

| 状態 | 意味 |
|---|---|
| 消灯 | リンクが確立されていません（ケーブル未接続、または接続先デバイスがオフ） |
| 緑の点灯 | リンクが確立されています（正常接続） |
| 緑の点滅 | データを送受信中です（アクティビティあり） |
| アンバーの点灯 | ポートが STP（スパニングツリープロトコル）によってブロックされています |
| 緑とアンバーの交互点滅 | リンク障害が検出されています |

<KeyPoint>
- ポート LED が消灯 → ケーブル未接続、接続先デバイスの電源オフ、またはポートが無効
- ポート LED がアンバー → STP によるブロッキング状態の可能性がある
- 緑とアンバーの交互点滅 → リンク障害（ケーブルの問題など）
</KeyPoint>

## モード LED とモードボタン

Cisco Catalyst スイッチには、ポート LED の表示モードを切り替えるための「Mode」ボタンがあります。このボタンを押すことで、ポート LED が示す情報を切り替えられます。

<KeyTerm term="Mode Button" reading="モードボタン">Cisco スイッチの前面パネルにあるボタンで、ポート LED の表示モードを切り替えます。STAT、DUPLX、SPEED、PoE などのモードがあります。</KeyTerm>

**主なモード:**

| モード | LED 名称 | ポート LED が示す情報 |
|---|---|---|
| STAT | ポートステータス | ポートの接続状態（デフォルト） |
| DUPLX | デュプレックス | 全二重（緑）/ 半二重（消灯） |
| SPEED | スピード | 10 Mbps（消灯）/ 100 Mbps（緑）/ 1000 Mbps（緑点滅） |
| PoE | Power over Ethernet | PoE の給電状態 |

デフォルトでは STAT モードが選択されており、ポートの接続状態が表示されます。Mode ボタンを押すたびにモードが切り替わり、対応するモード LED が点灯します。

## RPS LED

一部の Cisco スイッチには、RPS（Redundant Power Supply: 冗長電源装置）の状態を示す LED も搭載されています。

| 状態 | 意味 |
|---|---|
| 消灯 | RPS が接続されていません |
| 緑 | RPS が接続されており、バックアップ可能な状態です |
| アンバー | RPS がスタンバイモードまたは障害状態です |
| 緑の点滅 | RPS が内部電源の代わりに給電中です |

## PoE LED

PoE（Power over Ethernet）対応のスイッチでは、Mode ボタンで PoE モードに切り替えることで、各ポートの給電状態を確認できます。

<KeyTerm term="PoE" reading="ピーオーイー">Power over Ethernet の略。イーサネットケーブルを通じてネットワーク接続と同時に電力を供給する技術です。IP 電話、無線アクセスポイント、セキュリティカメラなどで使用されます。</KeyTerm>

**PoE モードでのポート LED:**

| 状態 | 意味 |
|---|---|
| 消灯 | PoE が無効、またはデバイスが PoE を要求していません |
| 緑 | PoE で正常に給電中です |
| アンバー | PoE が無効化されている、または給電に問題があります |
| 緑とアンバーの交互点滅 | PoE の給電が拒否されています（電力不足など） |

<KeyPoint>
- Mode ボタンで LED の表示モードを切り替え可能（STAT、DUPLX、SPEED、PoE）
- デフォルトは STAT モード（ポートステータス表示）
- PoE モードでは各ポートの電力供給状態を LED で確認できる
</KeyPoint>

## LED を使ったトラブルシューティング

LED インジケーターは、ネットワーク問題のトラブルシューティングにおいて最初に確認すべきポイントです。

1. **システム LED がアンバー**: デバイスのハードウェア障害の可能性があります。再起動やハードウェア交換を検討します。
2. **ポート LED が消灯**: ケーブルが正しく接続されているか、接続先デバイスの電源が入っているかを確認します。
3. **ポート LED がアンバー**: STP のブロッキング状態が原因の可能性があります。ネットワークトポロジーを確認します。
4. **すべてのポート LED がアンバーで点滅**: デバイスが POST を実行中（起動中）の可能性があります。しばらく待ちましょう。

## まとめ

- Cisco デバイスの LED は緑（正常）、アンバー（警告/エラー）、消灯（オフ/未接続）の 3 状態で動作状態を示します
- システム LED はデバイス全体の状態を、ポート LED は各ポートの接続状態を表します
- Mode ボタンを使って LED の表示モード（STAT / DUPLX / SPEED / PoE）を切り替えられます
- ポート LED がアンバーの場合、STP によるブロッキングの可能性があります
- LED の確認はトラブルシューティングの最初のステップとして重要です
