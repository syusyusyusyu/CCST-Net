## 概要

ネットワークに接続されるデバイスのうち、ネットワークの「端」に位置してユーザーやサービスと直接やり取りするデバイスを「エンドポイントデバイス」と呼びます。このトピックでは、PC、モバイルデバイス、IP 電話、プリンター、サーバー、IoT デバイスなど、さまざまなエンドポイントの種類と役割、ネットワーク接続の特徴について学びます。

## エンドポイントデバイスとは

<KeyTerm term="Endpoint Device" reading="エンドポイントデバイス">ネットワークの末端に位置し、ユーザーが直接操作するか、特定のサービスを提供するデバイスの総称です。</KeyTerm>

エンドポイントデバイスは、ネットワークインフラ機器（ルーター、スイッチなど）とは異なり、ネットワークの「利用者側」に位置するデバイスです。データの送信元または宛先となり、ネットワークサービスを利用したり提供したりします。

エンドポイントデバイスはネットワークにおけるセキュリティの観点でも重要です。攻撃者がネットワークに侵入する際の入口となりやすいため、適切な保護が必要です。

<KeyPoint>
- エンドポイントデバイスはネットワークの末端でデータを送受信するデバイスである
- ルーターやスイッチなどのインフラ機器とは区別される
- セキュリティ上の脆弱性となりやすいため、適切な保護が必要である
</KeyPoint>

## PC（パーソナルコンピュータ）

### デスクトップ PC

<KeyTerm term="Desktop PC" reading="デスクトップピーシー">据え置き型のパーソナルコンピュータ。有線 LAN（イーサネット）で接続されることが多く、高い処理能力を備えています。</KeyTerm>

デスクトップ PC は主にオフィスや自宅の固定された場所で使用されます。ネットワーク接続は通常、RJ-45 コネクタを使った有線イーサネット接続が用いられますが、USB Wi-Fi アダプターを使って無線接続することも可能です。

**ネットワーク設定:**
- NIC（ネットワークインターフェースカード）を搭載
- IP アドレスは DHCP で自動取得、または手動（スタティック）で設定
- 有線接続では安定した高速通信が可能

### ノートパソコン（ラップトップ）

ノートパソコンは有線と無線の両方の接続に対応しています。オフィスではドッキングステーションを通じて有線接続し、会議室やカフェでは Wi-Fi で接続するといった使い分けが一般的です。

### ワークステーション

ワークステーションは高い処理能力を持つ専門的な PC で、CAD 設計やビデオ編集、科学計算などに使用されます。大量のデータを扱うため、高速なネットワーク接続（10 Gbps イーサネットなど）が求められることがあります。

## モバイルデバイス

<KeyTerm term="Mobile Device" reading="モバイルデバイス">持ち運びが容易で、無線ネットワーク（Wi-Fi やセルラー）に接続するデバイス。スマートフォンやタブレットが代表例です。</KeyTerm>

### スマートフォン

スマートフォンは Wi-Fi とセルラー（4G/5G）の両方でネットワークに接続できます。テザリング機能を使って、スマートフォンをモバイルホットスポットとして他のデバイスにインターネット接続を共有することも可能です。

### タブレット

タブレットは Wi-Fi モデルとセルラー + Wi-Fi モデルがあります。画面が大きくキーボードを接続できるモデルもあり、ビジネス用途でも活用されています。

### その他のモバイルデバイス

- **スマートウォッチ**: Bluetooth で スマートフォンに接続、一部は Wi-Fi やセルラーにも対応
- **ポータブルゲーム機**: Wi-Fi でオンラインゲームに接続
- **電子書籍リーダー**: Wi-Fi で書籍のダウンロードや同期を行う

<KeyPoint>
- モバイルデバイスは主に Wi-Fi とセルラーで接続し、場所を選ばず利用できる
- テザリング機能により、スマートフォンを他のデバイスのインターネット接続手段として利用可能
- Bluetooth は短距離の PAN 接続に使われる
</KeyPoint>

## IP 電話（VoIP 電話）

<KeyTerm term="IP Phone" reading="アイピーフォン">インターネットプロトコル（IP）を使って音声通信を行う電話機。従来の電話回線ではなく、データネットワーク上で通話を実現します。</KeyTerm>

IP 電話は音声をデジタルデータに変換し、ネットワーク（LAN やインターネット）を通じて音声通話を行います。Voice over IP（VoIP）とも呼ばれる技術を使用しています。

**IP 電話の特徴:**
- イーサネットケーブルで LAN に接続（RJ-45）
- PoE（Power over Ethernet）対応の場合、データと電力を 1 本のイーサネットケーブルで供給可能
- PC パススルーポートを持つモデルもある（IP 電話の背面から PC に接続できる）
- 通話品質はネットワークの帯域幅とレイテンシに依存する
- Cisco の Unified Communications Manager（UCM）などで管理される

## ネットワークプリンター

<KeyTerm term="Network Printer" reading="ネットワークプリンター">ネットワークに直接接続され、複数のユーザーから印刷ジョブを受け付けることができるプリンターです。</KeyTerm>

ネットワークプリンターは、有線（イーサネット）または無線（Wi-Fi）でネットワークに接続します。ネットワーク上の複数のコンピュータから共有して使用できます。

**接続方式:**
- **有線（イーサネット）**: 安定した接続で、大量の印刷に適する
- **無線（Wi-Fi）**: ケーブル配線が不要で設置場所の自由度が高い
- **USB**: ネットワーク非対応のプリンターは USB で 1 台の PC に接続し、プリンター共有機能で他のユーザーに公開

多機能プリンター（MFP: Multi-Function Printer）は、印刷・コピー・スキャン・FAX の機能を備え、ネットワーク経由でこれらのサービスを提供します。

## サーバー

<KeyTerm term="Server" reading="サーバー">ネットワーク上のクライアント（PC やモバイルデバイスなど）にサービスを提供する高性能なコンピュータです。</KeyTerm>

サーバーは特定のサービスを提供するために設計された高性能なコンピュータです。通常、データセンターやサーバールームに設置され、高速かつ冗長なネットワーク接続が構成されます。

### サーバーの種類と役割

| サーバーの種類 | 役割 |
|---|---|
| Web サーバー | Web サイトのコンテンツを提供（HTTP/HTTPS） |
| メールサーバー | メールの送受信を処理（SMTP/IMAP/POP3） |
| DNS サーバー | ドメイン名と IP アドレスの変換を行う |
| DHCP サーバー | IP アドレスの自動割り当てを行う |
| ファイルサーバー | ファイルの共有ストレージを提供 |
| データベースサーバー | データベースの管理とクエリ処理を行う |
| 認証サーバー | ユーザー認証とアクセス制御を行う（Active Directory など） |

**サーバーのネットワーク特性:**
- 複数の NIC を搭載し、NIC チーミング（ボンディング）で冗長性と帯域幅を確保
- 高速接続（10 Gbps 以上）が必要な場合が多い
- 固定の IP アドレス（スタティック IP）が割り当てられる
- サーバールームではラックマウント型やブレード型が使用される

<KeyPoint>
- サーバーはクライアントにサービスを提供する高性能コンピュータである
- Web、メール、DNS、DHCP、ファイルなど、用途に応じた種類がある
- サーバーには固定 IP アドレスが割り当てられ、複数の NIC で冗長性を確保する
</KeyPoint>

## IoT デバイス

<KeyTerm term="IoT" reading="アイオーティー">Internet of Things（モノのインターネット）の略。センサーやアクチュエーターなど、従来はネットワークに接続されていなかった「モノ」がインターネットに接続される概念です。</KeyTerm>

IoT デバイスは、さまざまな「モノ」がネットワークに接続されてデータを収集・送信するデバイスです。

### IoT デバイスの例

- **スマートホーム**: スマートスピーカー、スマート照明、スマートサーモスタット、スマートロック
- **ウェアラブル**: フィットネストラッカー、スマートウォッチ
- **産業用**: 温度センサー、湿度センサー、監視カメラ（IP カメラ）
- **医療**: 患者モニタリングデバイス、スマート医療機器
- **インフラ**: スマートメーター（電力・ガス）、交通監視センサー

### IoT のネットワーク上の課題

- **大量のデバイス**: ネットワーク上のデバイス数が急増し、IP アドレスの管理が複雑化
- **セキュリティ**: 多くの IoT デバイスはセキュリティ機能が限定的で、攻撃の対象になりやすい
- **帯域幅**: 大量のデバイスが同時に通信すると、ネットワークの帯域幅を圧迫する
- **プロトコルの多様性**: MQTT、CoAP など、IoT 固有の軽量プロトコルが使われることがある

<KeyPoint>
- IoT デバイスはセンサーや家電など、さまざまな「モノ」をネットワークに接続する
- IoT デバイスの急増は IP アドレス管理やセキュリティの課題をもたらしている
- IoT デバイスのセキュリティは特に重要で、ネットワークのセグメント分離などの対策が必要である
</KeyPoint>

## まとめ

- **エンドポイントデバイス** はネットワークの末端でデータを送受信するデバイスの総称である
- **PC** は有線・無線で接続し、デスクトップは主に有線、ノート PC は状況に応じて使い分ける
- **IP 電話** はネットワーク経由で音声通信を行い、PoE で電力供給も可能である
- **サーバー** はクライアントにサービスを提供する高性能コンピュータで、固定 IP と冗長なネットワーク接続が必要である
- **IoT デバイス** はさまざまな「モノ」をネットワークに接続する技術であり、セキュリティ対策が特に重要である
